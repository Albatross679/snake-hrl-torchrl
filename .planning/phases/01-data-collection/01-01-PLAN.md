1. `log_event(event_log_path: Path, event_type: str, severity: str, worker_id: int | None = None, details: dict | None = None) -> None`
   - Appends a single JSON line to `event_log_path`.
   - Schema: `{"timestamp": "<ISO-8601 UTC>", "event_type": "<str>", "severity": "<info|warn|error>", "worker_id": <int|null>, "details": <dict>}`
   - Uses `datetime.datetime.now(datetime.timezone.utc).isoformat()` for timestamp.
   - Opens file in append mode, writes `json.dumps(event) + "\n"`, closes file. No buffering — each call is self-contained.
   - Event types used in Phase 1: `"worker_status"`, `"nan_discard"`, `"batch_saved"`, `"shutdown_requested"`, `"shutdown_complete"`, `"collection_started"`.

2. `validate_episode_finite(ep_data: dict) -> bool`
   - Returns `True` if all of `ep_data["states"]`, `ep_data["next_states"]`, and `ep_data["actions"]` pass `np.all(np.isfinite(...))`.
   - Returns `False` if any contain NaN or Inf.
   - Does NOT log or discard — caller handles that.
