  <name>Task 1: Data loading and analysis functions</name>
  <files>aprx_model_elastica/validate_data.py</files>
  <action>
Create `aprx_model_elastica/validate_data.py` with the following analysis functions. Do NOT use SurrogateDataset (it applies train/val split) — load all batch files directly to validate the FULL dataset.

**Data loading function `load_all_batches(data_dir: str) -> dict`:**
- Glob `data_dir/batch_*.pt`, load each with `torch.load(..., weights_only=True)`
- Concatenate all tensors: states (N, 124), actions (N, 5), next_states (N, 124), episode_ids (N,), step_indices (N,), serpenoid_times (N,)
- Offset episode_ids per file to make globally unique (same pattern as SurrogateDataset)
- Return dict with all concatenated tensors plus `n_batches` count

**Summary feature extraction `extract_summary_features(states: torch.Tensor) -> dict`:**
- CoM_x = states[:, POS_X].mean(dim=1)
- CoM_y = states[:, POS_Y].mean(dim=1)
- vel_mag = sqrt(states[:, VEL_X]^2 + states[:, VEL_Y]^2).mean(dim=1)
- mean_omega = states[:, OMEGA_Z].abs().mean(dim=1)
- Return dict mapping name -> (N,) tensor
- These 4 features match `compute_density_weights()` in dataset.py per user decision

**DVAL-01: Distribution analysis `analyze_distributions(data: dict) -> dict`:**
- Compute histograms for 4 summary features (use 50 bins)
- Compute histograms for 5 action dimensions (use 50 bins)
- For each histogram, compute: min, max, mean, std, skewness, kurtosis
- Return dict with per-feature stats and histogram data (bin_edges, counts)

**DVAL-02: Data quality checks `check_data_quality(data: dict) -> dict`:**
- NaN/Inf count: check states, next_states, actions for non-finite values. Report total count and rate.
- Duplicate transitions: compare (state, action) pairs. Use torch approach: hash rows by summing state*random_proj + action*random_proj, then count collisions. Report count and rate.
- Constant/near-constant features: for each of the 124 state dims + 5 action dims, compute std. Flag any with std < 1e-6 as constant. Report list of constant feature indices.
- Outlier detection: for each state dim, compute mean and std. Count values >5 sigma from mean. Report per-dim outlier count and total outlier rate.
- Return dict with all quality metrics

**DVAL-03: Temporal analysis `analyze_temporal(data: dict) -> dict`:**
- Episode length distribution: group by episode_id, count transitions per episode. Compute min/max/mean/median/std of episode lengths.
- Step index distribution: histogram of step_indices across all transitions. Compute early-vs-late bias: ratio of transitions in first quartile vs last quartile of step range.
- Return dict with episode_lengths array, step_histogram, bias_ratio

**DVAL-04: Action coverage `analyze_action_coverage(data: dict, n_bins: int = 20) -> dict`:**
- For each of the 5 action dimensions: bin into n_bins equal-width bins in [-1, 1]. Count samples per bin. Identify under-sampled bins (count < mean_count / 3).
- 5D joint coverage: create n_bins bins per dim, compute the flat bin index for each sample (same approach as compute_density_weights), count unique occupied bins vs total possible bins (n_bins^5). This is the "fill fraction."
- Per-dimension fill fraction (all bins with count > 0 / n_bins)
- Return dict with per_dim_histograms, per_dim_fill, joint_fill_fraction, under_sampled_regions (per dim: list of bin ranges)

All functions operate on CPU tensors. Use numpy only where torch doesn't have the needed function (e.g., skewness/kurtosis — use scipy.stats if available, else manual computation with torch).

**Report writer `write_report(all_results: dict, report_path: str)`:**
Write a structured markdown report to `data/surrogate/validation_report.md` with:

1. **Header**: "# Surrogate Data Validation Report" with timestamp and dataset summary (N transitions, N batches, N episodes)

2. **Pass/Fail Rubric Table**: A markdown table with columns: Metric | Value | Threshold | Status (PASS/WARN/FAIL). Metrics:
   - NaN/Inf rate: PASS if 0%, WARN if <0.1%, FAIL if >=0.1%
   - Duplicate rate: PASS if <0.1%, WARN if <1%, FAIL if >=1%
   - Constant features: PASS if 0, WARN if 1-3, FAIL if >3
   - Outlier rate (>5σ): PASS if <0.5%, WARN if <2%, FAIL if >=2%
   - Episode length CV: PASS if CV<0.5, WARN if CV<1.0, FAIL if CV>=1.0
   - Step index bias (Q1/Q4 ratio): PASS if 0.7-1.3, WARN if 0.5-1.5, FAIL otherwise
   - Action 5D fill fraction: PASS if >5%, WARN if >1%, FAIL if <=1%
   - Per-dim action fill: PASS if all dims >90%, WARN if all >70%, FAIL otherwise
   (Threshold values are at Claude's discretion per user decision — these are reasonable starting points)

3. **Distribution Analysis** section: Per-feature summary stats table (min, max, mean, std, skew, kurtosis). Reference figures.

4. **Data Quality** section: NaN/Inf details, duplicate count, constant features list, outlier breakdown by state group.

5. **Temporal Analysis** section: Episode length stats, step index bias ratio, reference figures.

6. **Action Coverage** section: Per-dim fill fractions, joint 5D fill fraction, under-sampled regions listed as prose. Reference figures.

7. **Recommendations** section: Prose recommendations based on results. Examples:
   - If action dim N has low fill, recommend targeted recollection for that range
   - If outlier rate high in velocity dims, recommend checking physics stability
   - If step index biased toward early steps, recommend longer episodes
   - Final go/no-go recommendation for surrogate training readiness
