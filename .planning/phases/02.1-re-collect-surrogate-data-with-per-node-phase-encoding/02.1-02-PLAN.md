```python
def collect_checkpoint_run(
    env: LocomotionElasticaEnv,
    action: np.ndarray,
    steps_per_run: int = 4,
) -> dict:
    """Collect one checkpoint run: run env.step(action) steps_per_run times.

    Each call to env.step() runs substeps_per_action=500 Elastica substeps internally.
    Returns states at each macro-step boundary: initial state + post-step states.

    Args:
        env: LocomotionElasticaEnv with rod already reset and perturbed (caller's
             responsibility to call env.reset() before this function).
        action: (5,) action array normalized in [-1, 1]. Same action for all steps.
        steps_per_run: Number of env.step() calls (default 4 → 5 states, 4 pairs).

    Returns:
        dict with:
            substep_states: (steps_per_run + 1, 124) float32 — state at each boundary
            action:         (5,) float32 — same action used for all steps
            t_start:        float — env._serpenoid._time at run start
            done_early:     bool — True if env signaled done before steps_per_run calls
    """
    t_start = float(env._serpenoid._time)

    # Capture initial state
    states = [RodState2D.pack_from_rod(env._rod)]  # state at boundary 0

    action_tensor = torch.tensor(action, dtype=torch.float32, device=env._device)
    td = env._reset()  # not called here — caller already reset. We need a td to call _step.
    # Actually: we need to build a td from scratch since env was reset by caller.
    # Create minimal td with action key for _step:
    td = env._reset()  # WRONG — don't re-reset. Build td differently.
    ...
```

**Important implementation note for executor:** The problem with calling env._step() is that
it requires a TensorDict with the current observation state. After env.reset(), env holds
the internal rod state, and we need to call _step N times without re-resetting. The correct
approach is:

```python
def collect_checkpoint_run(
    env: LocomotionElasticaEnv,
    action: np.ndarray,
    steps_per_run: int = 4,
) -> dict:
    """..."""
    from tensordict import TensorDict

    t_start = float(env._serpenoid._time)
    action_tensor = torch.tensor(action, dtype=torch.float32, device=env._device)

    # Capture initial state (env already reset by caller)
    states = [RodState2D.pack_from_rod(env._rod)]
    done_early = False

    # Build a minimal TensorDict for _step — observation is ignored by _step internally
    # (env._step only reads tensordict["action"]), so we pass a stub observation.
    td = TensorDict(
        {"action": action_tensor,
         "observation": torch.zeros(env.OBS_DIM, dtype=torch.float32, device=env._device)},
        batch_size=[],
        device=env._device,
    )

    for _ in range(steps_per_run):
        td["action"] = action_tensor
        td_result = env._step(td)
        states.append(RodState2D.pack_from_rod(env._rod))
        if td_result["done"].item():
            done_early = True
            break

    return {
        "substep_states": np.stack(states, axis=0),  # (K+1, 124) where K <= steps_per_run
        "action": action.copy(),                      # (5,)
        "t_start": t_start,
        "done_early": done_early,
    }
```

**Step C: Rewrite _collection_loop() to use checkpoint format**

Replace the episode-based inner loop with a run-based loop. Key changes:

1. Replace `collect_episode()` call with:
   - `env.reset()` + optional `perturb_rod_state()`
   - Sample one action (Sobol or random)
   - Call `collect_checkpoint_run(env, action, config.steps_per_run)`

2. Only accumulate runs where `not done_early` AND all states are finite (NaN check).
   Use `validate_run_finite(run_data)` — a new helper that checks substep_states for NaN/Inf:
   ```python
   def validate_run_finite(run_data: dict) -> bool:
       return np.all(np.isfinite(run_data["substep_states"]))
   ```

3. Batch accumulators change from list of 1D arrays to list of 2D arrays:
   - `batch_substep_states: List[np.ndarray]`  — each (K+1, 124)
   - `batch_actions: List[np.ndarray]`         — each (5,)
   - `batch_t_starts: List[float]`
   - `batch_episode_ids: List[int]`
   - `batch_run_ids: List[int]`

4. shared_counter increments by number of valid pairs: `len(run_data["substep_states"]) - 1`
   (typically 4, fewer if done_early).

5. episodes_per_save controls runs-per-save (not episodes). Keep the name to avoid
   breaking config — it now means "runs per save batch".
