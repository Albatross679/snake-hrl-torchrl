## Decisions Made

- **V2 format rationale:** The checkpoint format (5 boundary states) allows the dataset class to form any overlapping pairs `(s_i, a) → s_{i+1}` without losing data at episode boundaries. Phase 1 format stored only `(state, action, next_state)` pairs, which couldn't be extended to multi-step rollouts.
- **perturb_omega_std 0.05→1.5 rad/s:** Previous value (0.05) was far below operational range — serpentoid CPG produces omega_z ~1-10 rad/s. Perturbations need to cover the operational distribution.
- **steps_per_run=4 choice:** Balances coverage (more pairs per reset overhead) vs. state drift (4 steps = 2s sim time, rod stays in reasonable configuration).
- **Archive not delete:** data/surrogate_v1/ preserved for Phase 3 model comparison baseline if needed.
