## Decisions Made

- `get_sample_weights` uses `substep_states[:, :-1, :].reshape(-1, 124)` vectorized extraction (not naive loop) — O(N) vs O(N*K) loop; called once before training so either is acceptable
- Phase encoding dispatched through `encode_per_element_phase_batch()` with batch size 1 in `__getitem__` — consistent with training-time batched calls
- Components returned separately (state / per_element_phase / next_state / delta / action) — training code can apply independent normalization per component type
