## Decisions Made

- **--skip-disk-check at runtime:** The disk estimator uses 1 KB/transition but Phase 02.2 produces ~2 KB/transition (forces included), causing a false-positive rejection with 41 GB free disk. The script documents this and collection can be launched with `--skip-disk-check`. The disk pre-check should be updated in a future plan to use 2 KB/transition for force-inclusive collections.

- **W&B per-batch vs per-poll:** Single-process mode logs per batch save (in `_collection_loop()` via the `wb_run` parameter). Multi-process mode logs per health-check poll (in `_multiprocess_collect()` monitor loop) since W&B runs cannot be serialized across subprocess boundaries. Both include all 5 required metrics.

- **Manual stop at 10 GB:** User requested stopping collection manually at 10 GB rather than running to the 50M transition target, given available disk space (41 GB free). The 10 GB minimum requirement (RLDC-03) will be met with ~5M transitions.

## User Setup Required

None — W&B credentials are already in `~/.netrc`, collection is running.

To monitor collection:
```bash
tmux attach -t gsd-collect-rl
# or
tail -f output/collect_rl_step.log
```

To check data size:
```bash
python3 -c "
import os, torch, glob
files = glob.glob('data/surrogate_rl_step/batch_*.pt')
n = sum(torch.load(f, map_location='cpu', weights_only=True)['states'].shape[0] for f in files)
gb = sum(os.path.getsize(f) for f in files) / (1024**3)
print(f'{n:,} transitions, {gb:.2f} GB across {len(files)} files')
if gb >= 10.0:
    print('10 GB GUARANTEE MET — safe to stop collection')
"
```

To stop collection when 10 GB is reached:
```bash
tmux send-keys -t gsd-collect-rl:collect C-c
```
