  <action>
1. In `train_surrogate.py`, add three new CLI arguments to `parse_args()`:
   - `--hidden-dims`: comma-separated int list (e.g., "256,256,256"), default=None. When provided, override `config.model.hidden_dims` and recompute nothing (input_dim is always 131, output_dim is always 124).
   - `--run-name`: string, default=None. When provided and wandb is enabled, set `wandb.init(name=run_name, ...)`.
   - `--save-best-val-loss`: store_true, default False. When enabled, after training completes, write `best_val_loss` to `save_dir/metrics.json` as `{"best_val_loss": float, "final_epoch": int}`. This file is used by the sweep runner to pick the best run.

   In `main()`, after building config:
   - If `args.hidden_dims`: `config.model.hidden_dims = [int(x) for x in args.hidden_dims.split(",")]`
   - If `args.run_name` and `config.wandb_enabled`: pass `name=args.run_name` to `wandb.init()`
   - After training loop ends, if `args.save_best_val_loss`: write metrics.json to save_dir

2. Create `aprx_model_elastica/sweep.py` that:
   - Defines a list of sweep configs as dicts: `{"lr": float, "hidden_dims": str, "name": str}`
   - Per user decision, all 3 LR values and all 3 model sizes should be represented. Use 5 runs (within the 3-5 range per user decision) that cover the full grid intelligently:
     - lr=1e-4, hidden_dims=256,256,256, name=sweep_lr1e4_h256x3 (small model, low LR baseline)
     - lr=3e-4, hidden_dims=256,256,256, name=sweep_lr3e4_h256x3 (small model, default LR)
     - lr=3e-4, hidden_dims=512,512,512, name=sweep_lr3e4_h512x3 (medium model, default LR)
     - lr=3e-4, hidden_dims=512,512,512,512, name=sweep_lr3e4_h512x4 (large model, default LR)
     - lr=1e-3, hidden_dims=512,512,512, name=sweep_lr1e3_h512x3 (medium model, high LR)
   This covers: all 3 LRs (1e-4, 3e-4, 1e-3), all 3 hidden_dims options (256x3, 512x3, 512x4), with lr=3e-4 as the pivot connecting model sizes.
   - For each config, call `subprocess.run()` with:
     `python3 -m aprx_model_elastica.train_surrogate --lr {lr} --hidden-dims {hidden_dims} --save-dir output/surrogate/{name} --wandb --run-name {name} --epochs 200 --save-best-val-loss`
   - After all runs complete, read `metrics.json` from each run dir, print a summary table sorted by val loss, and identify the best run.
   - Use argparse with `--data-dir` (default "data/surrogate") and `--device` (default "auto") passthrough args forwarded to train_surrogate.
   - Print elapsed time per run and total time.
