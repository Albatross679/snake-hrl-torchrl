# Plan 03-04: Switch to CoM-Relative Velocities

## Goal

Extend the `raw_to_relative()` state representation to subtract center-of-mass velocity from per-node velocities. This addresses the root cause of poor velocity prediction (R2 = 0.065): absolute per-node velocities contain bulk locomotion motion (0-20 mm/s) that dominates the deformation signal (1-5 mm/s). The change is perfectly invertible (CoM velocity stored as 2 extra dims), increasing state dim from 128 to 130. All downstream code (configs, training loop R2 computation, validation) is updated to match.
