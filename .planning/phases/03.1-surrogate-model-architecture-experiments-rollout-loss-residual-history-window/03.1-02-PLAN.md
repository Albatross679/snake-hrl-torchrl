<objective>
Create aprx_model_elastica/arch_sweep.py following the sweep.py pattern and execute Experiments A (rollout loss weight tuning: 4 variants) and B (residual architecture: 1 variant). The sweep runner launches each config sequentially via subprocess, reads metrics.json after each run, and produces a ranked summary comparing all variants against the Phase 3 baseline (val_loss=0.2161).

Purpose: Generate the empirical evidence needed to select the best architecture for Phase 4 validation. The sweep is run inside the persistent tmux session `gsd-train` so training can proceed without blocking.

Output: arch_sweep.py + trained checkpoints in output/surrogate/arch_sweep/ + arch_sweep_summary.json.
</objective>
