<task type="auto">
  <name>Task 2: Per-scenario error aggregation and verdict logic</name>
  <files>aprx_model_elastica/validate.py, tests/test_validate_phase4.py</files>
  <action>
Build the scenario-level aggregation and PASS/WARN/FAIL verdict system on top of Task 1's infrastructure.

1. **ScenarioResult dataclass** to hold per-scenario aggregated metrics:
   ```python
   @dataclass
   class ScenarioResult:
       name: str
       episodes: int
       rmse_at_50: float          # mean RMSE@50 across episodes
       rmse_at_50_std: float
       com_drift_at_50: float     # mean CoM drift@50 (meters)
       com_drift_at_50_std: float
       real_displacement_at_50: float  # mean real CoM displacement (for relative drift)
       com_drift_relative: float  # com_drift / real_displacement (percentage)
       nan_episodes: int          # count of episodes with NaN
       per_component_rmse: dict   # {comp_name: mean_rmse_at_50}
       flagged_components: list   # components >2x average RMSE
       verdict: str               # "PASS", "WARN", "FAIL-SOFT", "FAIL-HARD"
       all_errors: list           # raw per-episode error dicts
   ```

2. **aggregate_scenario(errors_list, nan_steps_list)** function:
   - Compute mean/std of RMSE@50 and CoM drift@50 across episodes
   - Compute real CoM displacement at step 50 for relative drift
   - Identify flagged components (>2x mean RMSE across all components)
   - Count NaN episodes
   - Return ScenarioResult (verdict filled in separately)

3. **Verdict logic** — `assign_verdict(result: ScenarioResult, thresholds: dict)`:
   - If nan_episodes > 0: FAIL-HARD
   - CoM drift relative: PASS <10%, WARN 10-25%, FAIL-SOFT >25%
   - RMSE thresholds: data-driven, computed from random scenario as baseline
     - Run random scenario first, compute median RMSE@50
     - PASS < 2x median, WARN 2-5x median, FAIL > 5x median
   - Final verdict = worst of CoM and RMSE verdicts

4. **Overall verdict** — `compute_overall_verdict(scenario_results: dict)`:
   - If any FAIL-HARD: overall FAIL-HARD
   - If any FAIL-SOFT but some PASS: "PASS with caveats"
   - If all PASS or WARN: PASS
   - WARN is advisory, does not block

5. **Diagnosis section generator** — `generate_diagnosis(results: dict) -> str`:
   - Only generated when any scenario has FAIL verdict
   - Lists which scenarios failed, which components were flagged
   - Suggests likely causes and recommendations (more data, bigger model, longer training)

6. Add tests:
   - test_aggregate_scenario: correct mean/std computation
   - test_verdict_pass: CoM <10% and RMSE within threshold -> PASS
   - test_verdict_warn: CoM 10-25% -> WARN
   - test_verdict_fail_soft: CoM >25% -> FAIL-SOFT
   - test_verdict_fail_hard: NaN episodes -> FAIL-HARD
   - test_overall_mixed: some PASS some FAIL -> "PASS with caveats"
   - test_flagged_components: component >2x average gets flagged
