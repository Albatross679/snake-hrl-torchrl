<task type="auto">
  <name>Task 1: Figure generation and report writing</name>
  <files>aprx_model_elastica/validate.py, tests/test_validate_phase4.py</files>
  <action>
Add figure generation functions and structured report writer to validate.py. All figures use matplotlib Agg backend, dpi=150, bbox_inches="tight" per project convention.

**New figures (add to _save_plots or new save_validation_figures function):**

1. **Trajectory overlay plots** — `save_trajectory_overlays(scenario_results, save_dir)`:
   - For each scenario, select best/median/worst episode by RMSE@50
   - Plot real CoM (x,y) path in blue, surrogate CoM path in red dashed
   - One figure per episode = 3 per scenario, 4 scenarios = up to 12 figures (fewer if slow/fast gaits have 5 episodes)
   - Naming: `trajectory_overlay_{scenario}_{rank}.png` where rank in {best, median, worst}
   - Title includes scenario name, episode rank, and RMSE@50 value
   - CoM computed as mean of pos_x nodes and mean of pos_y nodes

2. **Per-component error heatmap** — `save_component_heatmap(scenario_results, save_dir)`:
   - Matrix: rows = 6 components (pos_x, pos_y, vel_x, vel_y, yaw, omega_z), cols = time steps (0 to 50)
   - Cell color = mean RMSE across all episodes and scenarios at that (component, step)
   - Use matplotlib imshow with 'hot' or 'YlOrRd' colormap
   - Naming: `component_error_heatmap.png`

3. **Scenario comparison bars** — `save_scenario_bars(scenario_results, save_dir)`:
   - Grouped bar chart: x-axis = scenarios, bars = RMSE@50 and CoM drift@50
   - Error bars showing std across episodes
   - Color-code bars by verdict (green=PASS, yellow=WARN, red=FAIL)
   - Naming: `scenario_comparison_bars.png`

4. **Keep existing plots** (RMSE over time, CoM drift over time) but update to show per-scenario colors with legend.

**Structured report** — `generate_report(scenario_results, overall_verdict, ckpt_dir, model_config, report_path)`:
   - Write markdown to `output/surrogate/validation_report.md` (or per-model if multiple models discovered)
   - Structure:
     ```
     # Surrogate Model Validation Report
     **Generated:** ISO timestamp
     **Checkpoint:** {ckpt_dir}
     **Architecture:** {model_config.arch}

     ## Summary
     | Scenario | Episodes | RMSE@50 | CoM Drift@50 | CoM Drift % | Verdict |
     |----------|----------|---------|--------------|-------------|---------|

     **Overall Verdict:** {overall_verdict}

     ## Per-Scenario Details
     ### {scenario_name}
     - Episodes: N, NaN episodes: M
     - RMSE@50: mean +/- std
     - CoM drift: mean +/- std (X% of real displacement)
     - Flagged components: [list]
     - Verdict: X

     ## Diagnosis (only if any FAIL)
     {diagnosis text}

     ## Figures
     - [list of generated figure paths]

     ## Thresholds Used
     - RMSE: PASS < X, WARN X-Y, FAIL > Y (derived from random scenario baseline)
     - CoM drift: PASS <10%, WARN 10-25%, FAIL >25%
     ```

**CLI wiring** — Update main() to:
   1. Parse args (keep existing, add --policy-checkpoint for PPO path, default to Session 10 best.pt)
   2. Discover checkpoints via discover_checkpoints(args.surrogate_checkpoint)
   3. For each discovered model: run all 4 scenarios, aggregate results, assign verdicts, generate figures, write report
   4. Print summary to stdout

Add test:
