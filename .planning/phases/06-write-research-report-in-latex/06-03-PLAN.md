**\subsection{Per-Element CPG Phase Encoding}** (~0.4 pages):
Explain the motivation: the target curvature $\kappa_i^\text{target}$ at element $i$ is a function of the element's arc-length position $s_i$ and the elapsed oscillation phase $\omega t$. A single global $(\sin(\omega t), \cos(\omega t))$ encoding (131-dim input) does not capture the spatial wave structure — each element experiences a different phase of the sinusoid. This limits surrogate accuracy, particularly for omega\_z (angular velocity), which has strong spatial structure.

Introduce the per-element phase encoding for element $e \in \{1, \ldots, N_e\}$:
\begin{equation}
  \boldsymbol{\phi}_e = \bigl(\sin(\omega t_e),\; \cos(\omega t_e),\; \kappa_e^\text{actual}\bigr)
\end{equation}
where $\omega t_e = 2\pi f \cdot t_e$ and $t_e$ is the effective time at element $e$'s arc-length position (computed from $k \cdot s_e / f$). Stacking $\boldsymbol{\phi}_e$ for all 20 elements yields a 60-dimensional feature vector. The full model input is:
\begin{equation}
  \mathbf{z}_t = \bigl[\bar{\mathbf{s}}_t \;\|\; \mathbf{a}_t \;\|\; \boldsymbol{\phi}_1, \ldots, \boldsymbol{\phi}_{N_e}\bigr] \in \mathbb{R}^{189}
\end{equation}
where $\bar{\mathbf{s}}_t$ is the z-score normalized state.

**\subsection{Surrogate Model Architecture}** (~0.4 pages):
Define the surrogate as a feedforward MLP approximating the ground-truth transition operator:
\begin{equation}
  f_\theta: \mathbf{z}_t \mapsto \overline{\Delta\mathbf{s}}, \quad \hat{\mathbf{s}}_{t+1} = \mathbf{s}_t + \sigma_\Delta \odot f_\theta(\mathbf{z}_t) + \mu_\Delta
\end{equation}
where $\mu_\Delta, \sigma_\Delta$ are the training-set mean and standard deviation of the state delta $(\mathbf{s}_{t+1} - \mathbf{s}_t)$, computed once before training. Explain the rationale for delta prediction: per-step changes are small relative to absolute state values, giving better numerical conditioning and acting as an implicit residual connection.

State the architecture: input layer $\mathbb{R}^{189}$, three hidden layers of 512 units each with LayerNorm and SiLU activation, output layer $\mathbb{R}^{124}$ initialized to zero (so the model starts by predicting zero delta). Total trainable parameters: approximately 659K.

**\subsection{Training Procedure}** (~0.5 pages):
Describe the two-phase training schedule:

Phase 1 (epochs 1--20): single-step MSE loss on normalized delta:
\begin{equation}
  \mathcal{L}_\text{single} = \frac{1}{N} \sum_{i=1}^{N} \bigl\| f_\theta(\mathbf{z}_t^{(i)}) - \overline{\Delta\mathbf{s}}^{(i)}_\text{true} \bigr\|^2
\end{equation}

Phase 2 (epoch 20 onward): combined loss with 8-step autoregressive rollout loss:
\begin{align}
  \mathcal{L} &= \mathcal{L}_\text{single} + \lambda_r \cdot \mathcal{L}_\text{rollout} \\
  \mathcal{L}_\text{rollout} &= \frac{1}{L} \sum_{t=0}^{L-1} \gamma^t \bigl\| \hat{\mathbf{s}}_{t+1} - \mathbf{s}_{t+1}^\text{true} \bigr\|^2
\end{align}
with $\lambda_r = 0.1$, rollout length $L = 8$, and $\gamma = 1$ (no discounting). Explain that the rollout loss penalizes autoregressive drift by back-propagating through the unrolled trajectory.

Describe density-weighted sampling: rare state configurations are upweighted via inverse density $w^{(i)} = 1/\hat{p}(\mathbf{c}^{(i)})$, where $\mathbf{c}^{(i)} = [\bar{x}_\text{CoM}, \bar{y}_\text{CoM}, \|\dot{\mathbf{x}}\|, |\bar{\omega}_z|]$ is a 4D summary feature. Weights are estimated from a joint histogram with 20 bins per dimension and clipped at 10$\times$ the mean weight. This compensates for Sobol clustering near the center of the action space.

State training hyperparameters from the best sweep configuration: learning rate 1e-3, Adam optimizer, batch size 4096, early stopping on validation MSE.

**\subsection{Physics Calibration Challenges}** (~0.7 pages):
Write a "Lessons Learned" narrative covering the physics calibration issues encountered during simulator setup. Use academic voice — frame these as empirical findings, not project anecdotes.

1. **Rod radius calibration**: The initial rod radius was set to $r = 0.001$ m (matching physical snake scale). This produced near-zero locomotion: bending stiffness $EI \propto r^4$, so at this scale the rod was effectively limp. Correcting to $r = 0.02$ m (with Young's modulus $E = 10^5$ Pa) restored serpentine motion. This illustrates a fundamental modeling challenge: in Cosserat rod simulation, the simulator's parameter regime must produce dynamics that match the physical phenomenon of interest, even if the absolute parameter values differ from physical measurements.

2. **Serpenoid wave direction sign error**: A sign error in the serpenoid curvature equation ($+k \cdot s$ instead of $-k \cdot s$) caused the propulsive wave to travel in the wrong direction, resulting in backward locomotion. This was discovered only after observing consistent negative forward velocity rewards across training runs. The fix required careful comparison of the CPG wave equation against the theoretical serpentine gait literature.

3. **Friction coefficient tuning**: The RFT friction parameters ($c_t, c_n$) required empirical tuning. Values derived from literature for soil-crawling snakes did not transfer to the dry-surface locomotion regime. Final values ($c_t = 0.01$, $c_n = 0.05$) were selected to produce stable propulsion across the range of CPG parameters in the action space.

Conclude with a synthesizing observation: surrogate model quality depends critically on the correctness of the underlying simulator. An accurate surrogate of an incorrectly calibrated physics model will replicate the physical errors.

**\subsection{Data Collection Pipeline Challenges}** (~0.6 pages):
Write a narrative covering the engineering challenges in the parallel data collection pipeline.

1. **Numba thread pool deadlock**: The first forward RFT computation call in each new worker process caused a deadlock inside Numba's internal thread pool. This manifested as a worker that would start, initialize its PyElastica environment, and then hang silently on the first physics step. Resolution: set \texttt{NUMBA\_NUM\_THREADS=1} in the worker's environment before any PyElastica imports. This eliminates thread contention at the cost of single-threaded Numba execution, which is acceptable because parallelism is achieved at the process level (16 workers) rather than within each worker.

2. **Stall detection false positives**: The health monitoring system tracked data output rate per worker. During initialization, workers legitimately produce no output for 10--30 seconds (environment construction, Python bytecode compilation). The stall detector initially triggered false alerts during this window, causing unnecessary worker respawns that compounded the startup overhead. Resolution: a startup grace period of 60 seconds before stall monitoring activates.

3. **Parallel scaling bottleneck**: Increasing from 16 to 40 workers produced slower total throughput, not faster. CPU profiling revealed that Python multiprocessing IPC overhead (sending 124-float state arrays across process boundaries via pickle) dominated at high worker counts. The optimal configuration on 48 CPUs was 16 workers — matching the available physical cores minus system overhead.

4. **omega\_z coverage gap**: Initial data collection with $\sigma_\omega = 0.05$ rad/s state perturbation produced a dataset where angular velocity was clustered near zero, far from the operational range of 1--10 rad/s encountered during actual CPG-driven locomotion. Surrogate accuracy on the omega\_z component was correspondingly poor. Resolution: increasing $\sigma_\omega$ to 1.5 rad/s for the V2 dataset provided the coverage needed for accurate omega\_z prediction.

Maintain academic voice throughout. Reference the specific issues by phenomenon, not by issue file names.
