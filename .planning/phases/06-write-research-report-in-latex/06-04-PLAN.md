Compile report/report.tex to PDF using Docker. LaTeX is not installed on this machine — use the Docker approach.

Run from the project root:
```bash
cd /home/coder/snake-hrl-torchrl
docker run --rm \
  -v $(pwd)/report:/workspace \
  -w /workspace \
  texlive/texlive:latest \
  latexmk -pdf -interaction=nonstopmode report.tex
```

If Docker is not available, try installing minimal TeX Live:
```bash
apt-get install -y texlive-latex-extra texlive-science texlive-fonts-recommended latexmk
cd /home/coder/snake-hrl-torchrl/report
latexmk -pdf report.tex
```

After compilation, check for:
1. LaTeX errors: grep for `^!` lines in the compilation log — these are fatal errors
2. Undefined references: grep for `LaTeX Warning: Reference` — these indicate missing \label{}
3. Undefined citations: grep for `Citation.*undefined` or `I couldn't find` in bibtex output — these produce [?] in the PDF

**Common fixes:**
- `Undefined control sequence \placeholder` → preamble definition missing; add `\newcommand{\placeholder}[1]{\textcolor{gray}{\textit{[Placeholder: #1]}}}`
- `Package natbib Error` → check natbib is loaded before hyperref
- `File not found` for figures → check `\graphicspath{{../figures/}}` is set; figures only included if they exist
- `Overfull \hbox` → warnings, not errors; acceptable for now
- Undefined citations (unknown BibTeX keys) → add missing entries to references.bib

If there are compilation errors, fix them in report/report.tex or report/references.bib and re-compile until the PDF builds successfully.

The PDF should be approximately 10-15 pages at this stage (skeleton + Background + Related Work + Methods + Discussion physics/pipeline sections).
