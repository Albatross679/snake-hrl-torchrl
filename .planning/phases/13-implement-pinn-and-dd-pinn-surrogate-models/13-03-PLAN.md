Create `src/pinn/train_regularized.py` as a standalone training script:

**Entry point:** `python -m src.pinn.train_regularized`

**CLI args:**
- `--data-dir` (default: "data/surrogate_rl_step")
- `--lambda-phys` (default: 0.01, float)
- `--epochs` (default: 200, reduced from 999999 for sweep)
- `--lr` (default: 1e-3, matching Phase 3 best)
- `--hidden-dims` (default: "512,512,512", matching Phase 3 best)
- `--use-residual` (flag, default False)
- `--no-wandb` (flag)
- `--run-name` (optional)
- `--save-dir` (default: "output/surrogate/pinn_regularized")
- `--device` (default: "auto")
- `--use-relobralo` (flag, default True)
- `--curriculum-warmup` (default: 0.2, fraction of epochs for data-only warmup)
- `--patience` (default: 30)

**Training loop modifications vs train_surrogate.py:**
1. Load data with FlatStepDataset (same as Phase 3)
2. Convert to relative 130-dim state (same as Phase 3)
3. Fit StateNormalizer (same as Phase 3)
4. Build model (SurrogateModel or ResidualSurrogateModel based on --use-residual)
5. **NEW: Create PhysicsRegularizer(dt=0.5)**
6. **NEW: Create ReLoBRaLo(n_losses=2) if --use-relobralo**
7. Training loop:
   - `loss_data = F.mse_loss(model(state_norm, action, time_enc), delta_norm_target)`
   - **NEW:** Compute physics loss on RAW state (need to keep raw state alongside relative):
     - `raw_state` = original 124-dim state (kept from dataset loading)
     - `delta_pred_raw` = `normalizer.denormalize_delta(model_output)` then convert back to raw via relative_to_raw
     - Actually simpler: physics regularizer operates on raw deltas directly
     - `delta_pred_denorm = normalizer.denormalize_delta(model_output)` -- this is 130-dim relative delta
     - To get raw delta: need to track that delta in relative space maps to delta in raw space
     - **Approach:** Keep raw_state from dataset. Compute predicted raw_next = relative_to_raw(raw_to_relative(raw_state) + delta_pred_denorm). Then delta_pred_raw = predicted_raw_next - raw_state.
     - `loss_phys = regularizer(raw_state, delta_pred_raw)`
   - **NEW:** Curriculum: `phys_weight = get_curriculum_weight(epoch, total_epochs, max_weight=lambda_phys, warmup_frac=curriculum_warmup)`
     - 0 for first `warmup_frac` of epochs, linear ramp to `max_weight` over next 30%, then hold
   - **NEW:** If ReLoBRaLo: `weights = balancer.update([loss_data, phys_weight * loss_phys])`; `loss = weights[0] * loss_data + weights[1] * phys_weight * loss_phys`
   - Else: `loss = loss_data + phys_weight * loss_phys`
8. **NEW:** W&B logging: log loss_data, loss_phys, phys_weight, relobralo_weight_data, relobralo_weight_phys per epoch
   - W&B project: "snake-hrl-pinn" (separate from "snake-hrl-surrogate")
9. Same early stopping, checkpoint saving, cosine LR as train_surrogate.py
10. **NEW:** At end of training, compute per-component RMSE on validation set:
    - Components: pos_x, pos_y (mm), vel_x, vel_y (mm/s), yaw (rad), omega_z (rad/s)
    - Save to `{save_dir}/eval_metrics.json`
    - Print summary table to console

**Sweep script** (at bottom of file or separate function):
- `--sweep` flag runs all 4 lambda values sequentially: {0.001, 0.01, 0.1, 1.0}
- Each run saves to `{save_dir}/lambda_{value}/`
- After sweep, generate comparison plots

**Post-sweep evaluation:**
