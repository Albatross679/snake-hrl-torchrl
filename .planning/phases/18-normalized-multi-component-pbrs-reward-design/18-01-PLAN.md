# Plan 18-01: Normalized Multi-Component PBRS Reward Design

## Goal

Refactor the Choi 2025 reward system to use a **uniform normalized reward architecture** where:
1. All non-PBRS reward components are normalized to known ranges before weighting
2. PBRS is decomposed into per-objective potentials (distance, heading, smoothness)
3. Weights become pure importance coefficients, not scale-entangled knobs
4. Per-component decomposed rewards are logged for diagnostics

## Scope

- **Files modified:** `papers/choi2025/rewards.py`, `papers/choi2025/config.py`, `papers/choi2025/env.py`, `papers/choi2025/train_ppo.py`
- **Files added:** None (refactor existing)
- **Tests:** `tests/test_choi2025.py` updated
- **Does NOT touch:** SAC trainer, MM-RKHS trainer, obstacle tasks (those stay as-is for now)

## Architecture

```
total_reward = Σ wᵢ · normalize(rᵢ)     ← base components, normalized then weighted
             + Σ (γ·Φⱼ(s') - Φⱼ(s))     ← PBRS components, raw (naturally centered)
```

### Reward Components (follow_target task)

| Component | Type | Raw range | Normalized range | Default weight |
|-----------|------|-----------|-----------------|----------------|
| Distance | Base | [0, 1] | [0, 1] (already) | 1.0 |
| Heading alignment | Base | [0, 1] | [0, 1] (already) | 0.0 (off) |
| Action smoothness | Penalty | [-action_dim, 0] | [-1, 0] | 0.0 (off) |
| Distance PBRS | Shaping | ~[-0.1, 0.1] | Raw (unweighted) | N/A |
| Heading PBRS | Shaping | ~[-0.1, 0.1] | Raw (unweighted) | N/A |

### Key Design Decisions

1. **Improvement bonus removed.** It duplicates PBRS (both measure prev_dist - dist) but without the policy-invariance guarantee. PBRS replaces it entirely.
2. **PBRS terms are unweighted.** They guide the agent without changing the optimal policy. No user knobs needed.
3. **Action smoothness as PBRS.** Φ(s) = -||a||²/action_dim. This preserves the optimal policy while still guiding toward smooth actions.
4. **Workspace radius normalization for distance PBRS.** Φ(s) = -dist/workspace_radius bounds the potential to [-1, 0].

## Tasks

### Task 1: Refactor `rewards.py` — normalized component architecture

**File:** `papers/choi2025/rewards.py`

**Changes to `compute_follow_target_reward`:**

1. Remove `improvement_weight` / `improvement_reward` logic entirely (lines 81-83)
2. Normalize action smoothness: `smooth = -np.sum(delta**2) / (2 * action_dim)` → range [-1, 0]
3. Add workspace_radius parameter for distance PBRS normalization
4. Restructure into clear sections: base components → weighted sum → PBRS addition
5. Add heading PBRS: Φ = cos_similarity(tip_tangent, to_target_dir)
6. Return all components (base + PBRS) in the components dict

**New signature:**
```python
def compute_follow_target_reward(
    tip_pos, target_pos, prev_tip_pos,
    tip_tangent=None, prev_tip_tangent=None,
    heading_weight=0.0,
    smooth_weight=0.0,
    prev_dist=None,
    pbrs_gamma=0.0,
    action=None, prev_action=None,
    action_dim=10,
    workspace_radius=1.0,
    return_components=False,
) -> float | tuple[float, dict]:
```

**Key logic:**
```python
# === Base components (normalized, weighted) ===
dist_reward = np.exp(-5.0 * dist)                    # [0, 1]
heading_reward = (1.0 + cos_sim) / 2.0               # [0, 1]
smooth_penalty = -np.sum(delta**2) / (2 * action_dim) # [-1, 0]

total = (1.0 * dist_reward                           # always on
       + heading_weight * heading_reward
       + smooth_weight * smooth_penalty)

# === PBRS (raw, unweighted, naturally centered) ===
if pbrs_gamma > 0 and prev_dist is not None:
    phi_dist = -dist / workspace_radius
    phi_dist_prev = -prev_dist / workspace_radius
    f_dist = pbrs_gamma * phi_dist - phi_dist_prev
    total += f_dist

    if tip_tangent is not None and prev_tip_tangent is not None:
        phi_head = cos_sim
        phi_head_prev = prev_cos_sim
        f_head = pbrs_gamma * phi_head - phi_head_prev
        total += f_head
```

**Removed parameters:** `pbrs_only`, `improvement_weight`

- [ ] PBRS terms satisfy F = γΦ(s') - Φ(s) form exactly

1. **Breaking existing training runs.** Mitigation: removed parameters had default=0 (disabled), so default behavior is unchanged. Only the improvement bonus (if someone was using it) is gone.
2. **PBRS heading requires prev_tip_tangent.** Mitigation: only computed when `heading_weight > 0` AND `pbrs_gamma > 0`, both off by default.
3. **Obstacle reward normalization changes magnitudes.** Mitigation: Task 6 is last and adjusts defaults to preserve similar effective magnitudes.
