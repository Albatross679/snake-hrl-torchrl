   Present the PDE -> ODE -> arithmetic map pipeline:
   - Stage 1 (spatial discretization): PDE in $(s,t)$ becomes ODE system in $t$ via method of lines. Result: $\dot{\mathbf{s}} = \mathbf{g}(\mathbf{s}, \mathbf{a}, t)$ where $\mathbf{g}$ is a closed-form function (geometry -> strains -> constitutive law -> forces -> accelerations).
   - Stage 2 (explicit time integration): ODE $\dot{\mathbf{s}} = \mathbf{g}(\mathbf{s})$ becomes arithmetic map $\mathbf{s}^{n+1} = \Phi(\mathbf{s}^n)$. Position Verlet evaluates $\mathbf{g}$ at the current state and updates directly — the unknown $\mathbf{s}^{n+1}$ appears only on the left-hand side.
   Use a tabular or description-list format.

4. **Explicit vs implicit definition (compact table):**
   A small 3-column table (Method | Update rule | Type) with three rows:
   - Forward Euler: $\mathbf{s}^{n+1} = \mathbf{s}^n + \Delta t\,\mathbf{g}(\mathbf{s}^n)$ | Explicit
   - Backward Euler: $\mathbf{s}^{n+1} = \mathbf{s}^n + \Delta t\,\mathbf{g}(\mathbf{s}^{n+1})$ | Implicit
   - Position Verlet (Elastica): Eqs. in Algorithm 1 | Explicit
   Then 1 sentence: "The distinction is whether $\mathbf{s}^{n+1}$ appears on the right-hand side — if it does, a Newton solve is required (see \cref{sec:physics:dismech:implicit})."

   The neural surrogate replaces the 500-step composition $T(\mathbf{s}_t, \mathbf{a}_t) = V_{\Delta t}^{500}(\mathbf{s}_t; \mathbf{a}_t)$ — a deterministic sequential map, not a PDE solver. Each substep $V_{\Delta t}$ is a chain of explicit arithmetic (geometry, constitutive law, force balance, Verlet update). The surrogate collapses this sequential composition into a single forward pass. Because $T$ is smooth and deterministic (no iterative convergence), a feedforward network can approximate the composed map directly.
