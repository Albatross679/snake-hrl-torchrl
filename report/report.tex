\begin{abstract}
Reinforcement learning for soft and continuum robots is bottlenecked by the cost of first-principles simulation: a Cosserat rod snake robot requires ${\sim}$500 substeps per control step, costing ${\sim}$46\,ms per environment transition.
This report investigates two directions for enabling practical RL-based control of a soft manipulator: neural surrogate modeling to overcome simulation cost, and reward architecture design for effective policy learning.
On the surrogate side, a data-driven MLP trained on 4.3M Elastica transitions achieves centre-of-mass $R^2 = 0.852$ in single-step prediction, targeting a four-to-five order-of-magnitude inference speedup, though velocity prediction remains weak ($R^2 = 0.145$); physics-informed extensions (PINN, DD-PINN) are proposed as future work.
On the reward side, systematic RL experiments on DisMech reveal that reward design and curriculum learning dominate performance: a reward-shaping ablation across five variants shows a 52\% range in the fair distance metric with identical hyperparameters, and curriculum learning alone accounts for 16\% improvement over the vanilla baseline.
Extended training to 50M frames confirms these are fundamental reward-landscape limitations, not sample-budget issues: the $\exp(-5d)$ reward is essentially zero at operating distances ($d \approx 0.6$), creating a hard learning plateau.
A normalized multi-component reward architecture with per-objective potential-based reward shaping (PBRS) decouples component scale from importance weights, achieving a best distance of $d = 0.543$ ($-$24\% vs.\ baseline) and enabling detection of reward hacking through per-component logging.
We also document SAC gradient instability without entropy regularization (actor gradient norms growing to $10^{10}$ over 20M frames).
\end{abstract}

\section*{Glossary}
\label{sec:glossary}
\addcontentsline{toc}{section}{Glossary}

\begin{description}[style=nextline, leftmargin=3.2cm, labelwidth=3cm, font=\normalfont\bfseries]

  \item[Cosserat rod]
    A continuum mechanics model for slender elastic bodies that tracks both position and orientation (director frames) along the rod centreline, capturing bending, torsion, shear, and extension.

  \item[PyElastica]
    An open-source Cosserat rod simulator using explicit symplectic (Position Verlet) time integration. Used in this work for data collection and direct RL training.

  \item[DisMech]
    A Cosserat rod simulator using implicit backward Euler time integration. Used for the Choi2025 follow-target RL experiments.

  \item[CPG]
    \emph{Central Pattern Generator.} A bio-inspired oscillator that produces rhythmic motor commands. The snake robot's action space consists of five normalised CPG parameters (amplitude, frequency, spatial wavenumber, phase offset, lateral bias).

  \item[Serpenoid curve]
    The travelling-wave body shape $\kappa(s,t) = A\sin(2\pi(ks - ft) + \phi_0)$ that produces serpentine locomotion; parameterised by CPG actions.

  \item[Surrogate model]
    A neural network $f_\theta$ trained to approximate the physics simulator's state transition, replacing an expensive first-principles solver with a single GPU forward pass.

  \item[MLP]
    \emph{Multi-Layer Perceptron.} A feedforward neural network of fully connected layers. The implemented surrogate uses three hidden layers of 512 units with LayerNorm and SiLU activations.

  \item[PINN]
    \emph{Physics-Informed Neural Network.} A neural network whose training loss includes a residual term derived from the governing PDEs, encouraging physics-consistent predictions. Proposed but not implemented in this work.

  \item[DD-PINN]
    \emph{Domain-Decoupled Physics-Informed Neural Network.} A PINN variant where the network outputs parameters of an analytically differentiable ansatz rather than the state itself, eliminating autodiff cost and satisfying initial conditions exactly. Proposed but not implemented in this work.

  \item[RFT]
    \emph{Resistive Force Theory.} An anisotropic friction model that decomposes ground contact forces into tangential and normal components, widely used for limbless locomotion on flat surfaces.

  \item[RL]
    \emph{Reinforcement Learning.} A learning paradigm where an agent interacts with an environment to maximise cumulative reward.

  \item[PPO]
    \emph{Proximal Policy Optimisation.} An on-policy actor-critic algorithm that constrains policy updates via a clipped surrogate objective, balancing sample efficiency with training stability.

  \item[SAC]
    \emph{Soft Actor-Critic} \citep{Haarnoja2018}. An off-policy actor-critic algorithm that maximises a combined reward-plus-entropy objective, encouraging exploration and robust policies.

  \item[GAE]
    \emph{Generalised Advantage Estimation.} A variance-reduction technique for policy gradient methods that exponentially weights multi-step temporal-difference errors via a parameter $\lambda$.

  \item[PBRS]
    \emph{Potential-Based Reward Shaping.} A reward augmentation $F(s,s') = \gamma\,\Phi(s') - \Phi(s)$ that preserves optimal-policy invariance while providing denser learning signal. Used per-objective in the normalised reward architecture.

  \item[Curriculum learning]
    A training strategy that gradually increases task difficulty. In this work, the target spawn distance starts close and expands over training, ensuring the agent encounters reward-dense regions early.

  \item[HRL]
    \emph{Hierarchical Reinforcement Learning.} A framework that decomposes control into multiple levels (e.g.\ meta-controller, approach policy, coil policy) to handle temporally extended tasks.

  \item[TorchRL]
    A PyTorch-native RL library providing composable environment wrappers, replay buffers, and loss modules. Used as the training framework throughout this project.

  \item[CoM]
    \emph{Centre of Mass.} The mass-weighted average position of all discretisation nodes; used as a summary statistic for locomotion evaluation.

  \item[CFL condition]
    \emph{Courant--Friedrichs--Lewy condition.} A numerical stability constraint relating time-step size to spatial resolution and wave speed in explicit integrators.

\end{description}

\section{Introduction}
\label{sec:intro}

Reinforcement learning for soft and continuum robots is bottlenecked by the cost of first-principles simulation.
A snake robot modeled as a Cosserat rod requires resolving elastic body dynamics, anisotropic ground friction, and internally generated muscle moments at every simulation substep --- PyElastica \citep{Naughton2021} uses 500 Position Verlet substeps per control step, costing ${\sim}46$\,ms per environment step.
Comprehensive surveys of soft robot modeling \citep{Armanini2022} and model-based control \citep{DellaSantina2023} highlight that this computational burden remains a central bottleneck.
This report presents a neural surrogate $f_\theta \approx T$ that replaces the transition operator with a single GPU forward pass (${\sim}0.001$\,ms), targeting a four-to-five order-of-magnitude speedup for RL training.

The project has two main contributions:
\begin{enumerate}
  \item \textbf{Neural surrogate modeling} --- a data-driven MLP surrogate trained on 4.3M Elastica transitions, with physics-regularized and DD-PINN extensions proposed as future directions (\Cref{sec:surrogate}).
  A direct-simulation RL baseline on Elastica validates the environment pipeline (\Cref{sec:rl-elastica}).
  \item \textbf{Reward design for soft manipulator control} --- systematic RL experiments on DisMech reveal that reward function and curriculum learning dominate performance; a normalized multi-component architecture with per-objective PBRS addresses the identified structural problems (\Cref{sec:experiments}).
\end{enumerate}

Two physics backends are investigated --- PyElastica (explicit symplectic) and DisMech (implicit backward Euler).
The shared simulation framework is described in \Cref{sec:system}.

% ---- 3.1 Cosserat Rod Formulation ----
\subsection{Cosserat Rod Formulation}
\label{sec:system:cosserat}

The system is fully specified by the tuple $\mathcal{S} = (\mathcal{L},\; \Omega,\; \text{BC},\; \text{IC},\; \boldsymbol{\theta})$:

\begin{table}[H]
  \centering
  \small
  \renewcommand{\arraystretch}{1.3}
  \caption{Well-posedness tuple for the Elastica Cosserat rod system.}
  \label{tab:system-tuple}
  \begin{tabular}{clp{8.5cm}}
    \toprule
    Component & Name & Specification \\
    \midrule
    $\mathcal{L}$ & Governing equations &
      Cosserat rod PDEs: conservation of linear and angular momentum with linear elastic constitutive law \\
    $\Omega$ & Domain &
      $s \in [0, L]$, $L = 0.5$\,m (arc length) $\times$ $t \in [0, T]$ (time) \\
    BC & Boundary conditions &
      Free--free: $\mathbf{F} = \mathbf{M} = \mathbf{0}$ at $s = 0$ and $s = L$ \\
    IC & Initial conditions &
      Straight rod at rest: $\mathbf{x}(s,0) = \mathbf{x}_0 + s\hat{\mathbf{e}}_x$,\; $\mathbf{v} = \boldsymbol{\kappa} = \boldsymbol{\omega} = \mathbf{0}$ \\
    $\boldsymbol{\theta}$ & Parameters &
      $E{=}10^5$\,Pa, $\nu{=}0.5$, $\rho{=}1200$\,kg/m$^3$, $R{=}0.02$\,m, $c_t{=}0.01$, $c_n{=}0.05$, $\gamma{=}0.002$ \\
    Control & CPG forcing &
      $\mathbf{a} = (A, f, k, \phi_0, b) \in \mathbb{R}^5$ enters as time-varying rest curvature $\boldsymbol{\kappa}_0(s,t)$ \\
    \bottomrule
  \end{tabular}
\end{table}

The governing PDEs are:
\begin{subequations}
\label{eq:cosserat-pdes}
\begin{alignat}{2}
  \rho A \,\frac{\partial \mathbf{v}}{\partial t}
    &= \frac{\partial \mathbf{F}}{\partial s} + \mathbf{f}_{\text{ext}}
    &\qquad& \text{(linear momentum)}
  \label{eq:pde-linear} \\[4pt]
  \rho \mathbf{I} \,\frac{\partial \boldsymbol{\omega}}{\partial t}
    &= \frac{\partial \mathbf{M}}{\partial s}
      + \mathbf{m}_{\text{ext}}
      + \frac{\partial \mathbf{x}}{\partial s} \times \mathbf{F}
    &\qquad& \text{(angular momentum)}
  \label{eq:pde-angular}
\end{alignat}
\end{subequations}
with constitutive closure:
\begin{equation}
  \mathbf{F} = \mathbf{S}\,(\boldsymbol{\varepsilon} - \boldsymbol{\varepsilon}_0), \quad
  \mathbf{M} = \mathbf{B}\,(\boldsymbol{\kappa} - \boldsymbol{\kappa}_0), \quad
  \mathbf{S} = \operatorname{diag}(GA, GA, EA), \quad
  \mathbf{B} = \operatorname{diag}(EI, EI, GJ)
  \label{eq:constitutive}
\end{equation}

Spatial discretization on a staggered grid ($N{=}21$ nodes, $N_e{=}20$ elements) reduces the PDE to 124 coupled first-order ODEs:
\begin{equation}
  \dot{\mathbf{s}} = \mathbf{g}(\mathbf{s},\, \mathbf{a},\, t), \qquad
  \mathbf{s} \in \mathbb{R}^{124},\;\; \mathbf{a} \in \mathbb{R}^{5}
  \label{eq:ode-system}
\end{equation}

Position Verlet ($\Delta t = 0.001$\,s, $n_{\text{sub}} = 500$ substeps per RL step of $\Delta t_{\text{ctrl}} = 0.5$\,s) produces the transition operator:
\begin{equation}
  \mathbf{s}_{t+1} = T(\mathbf{s}_t,\, \mathbf{a}_t)
    = \underbrace{V_{\Delta t}(\,\cdot\,;\, \boldsymbol{\kappa}^{\text{rest},499}) \circ \cdots \circ V_{\Delta t}(\,\cdot\,;\, \boldsymbol{\kappa}^{\text{rest},0})}_{500 \text{ substeps, each with distinct rest curvature}}(\mathbf{s}_t)
  \label{eq:transition-operator}
\end{equation}

See \Cref{app:physics} for detailed PDE derivations, staggered grid operators, external force handling, and the DisMech formulation.

\Cref{alg:elastica-transition} expands the full Elastica transition operator.

\begin{algorithm}[ht!]\small
\caption{Elastica Transition Operator $T(\mathbf{s}_t,\, \mathbf{a}_t,\, t_{\text{step}}) \to \mathbf{s}_{t+1}$}\label{alg:elastica-transition}
\begin{algorithmic}[1]
\Require State $\mathbf{s}_t \in \mathbb{R}^{124}$, action $\mathbf{a}_t \in \mathbb{R}^5$, step time $t_{\text{step}}$
\Ensure Next state $\mathbf{s}_{t+1} \in \mathbb{R}^{124}$
\State $\mathbf{s} \gets \mathbf{s}_t$
\For{$n = 0, 1, \ldots, 499$}
  \State $t^n \gets t_{\text{step}} + n \cdot \Delta t$
  \State Update rest curvature: $\kappa_v^{\text{rest}} \gets A \sin(2\pi k \cdot s_v + 2\pi f \cdot t^n + \phi_0) + b$ for $v = 1, \ldots, 19$
  \State Verlet half-step: $\mathbf{q}^{1/2} \gets \mathbf{q} + \tfrac{1}{2}\Delta t\;\dot{\mathbf{q}}$
  \State Compute geometry: edge vectors, tangents, strain, curvature
  \State Compute internal forces (shear--stretch) and torques (bending)
  \State Compute external forces: RFT friction + damping + gravity
  \State Compute accelerations: $\ddot{\mathbf{x}} \gets (\mathbf{F}^{\text{int}} + \mathbf{F}^{\text{ext}}) / \mathbf{m}$
  \State Verlet velocity update + full-step position update
\EndFor
\State \textbf{return} $\mathbf{s}_{t+1} = (\mathbf{q},\, \dot{\mathbf{q}})$
\end{algorithmic}
\end{algorithm}

\noindent
\textit{Cost:} ${\sim}377{,}000$ FLOPs/step (${\sim}46$\,ms on CPU).
The CPG phase advances by $\Delta\phi = 2\pi f \Delta t$ per substep --- the surrogate must learn the \textit{integrated effect} of the traveling wave across 500 distinct curvature configurations.

% ---- 3.2 CPG Control ----
\subsection{CPG Control}
\label{sec:system:cpg}

The Central Pattern Generator drives locomotion by imposing time-varying rest curvature on the rod.
When the rest curvature differs from the rod's current curvature, elastic bending moments drive the rod toward the target shape.
As the CPG sweeps a sinusoidal curvature wave along the body, anisotropic ground friction (higher normal than tangential) converts the lateral undulation into net forward thrust---the same mechanism underlying biological serpentine locomotion.
The RL agent selects five CPG wave parameters per control step:

\begin{table}[H]
  \centering
  \small
  \caption{Action vector $\mathbf{a}_t \in \mathbb{R}^{5}$: CPG wave parameters.}
  \label{tab:action-vector}
  \begin{tabular}{clrl}
    \toprule
    Symbol & Name & Range & Units \\
    \midrule
    $A$      & Amplitude    & $[0, 5]$      & rad\,m$^{-1}$ \\
    $f$      & Frequency    & $[0.5, 3.0]$  & Hz \\
    $k$      & Wave number  & $[0.5, 3.5]$  & wavelengths/body \\
    $\phi_0$ & Phase offset & $[0, 2\pi]$   & rad \\
    $b$      & Turn bias    & $[-2, 2]$     & rad\,m$^{-1}$ \\
    \bottomrule
  \end{tabular}
\end{table}

\noindent
The serpenoid rest curvature at each Voronoi node:
\begin{equation}
  \kappa_v^{\text{rest}}(t) = A \sin(2\pi k \cdot s_v + 2\pi f \cdot t + \phi_0) + b, \qquad v = 1, \ldots, 19.
  \label{eq:serpenoid}
\end{equation}

    \multicolumn{3}{l}{\textit{Supported forces}} \\

    Self-contact             & ---                         & \checkmark (FCL) \\
    \midrule
    \multicolumn{3}{l}{\textit{Control}} \\
    Curvature mechanism      & Elastic rest shape ($\kappa_0$) & Hard constraint (natural strain) \\
    CPG resolution           & 500 curvature updates/step  & 1 curvature snapshot/step \\
    \midrule
    \multicolumn{3}{l}{\textit{Metadata}} \\
    Theory                   & Cosserat rod                            & 2008 DER \citep{Bergou2008}; 2010 \citep{Bergou2010} \\

  \end{tabular}
\end{table}

The fundamental trade-off: PyElastica's 500 substeps capture continuous CPG evolution but cost $4\times$ more wall-clock time.
DisMech's single implicit step is faster and unconditionally stable, but collapses intra-step dynamics.
For surrogate modeling, Elastica presents a harder approximation target.

The DisMech ecosystem has evolved significantly beyond the original DER formulation.
\citet{Choi2024dismech} introduced DisMech as a full simulator with implicit contact handling and real2sim validation, achieving an order-of-magnitude speedup over prior DER implementations.
\citet{Lahoti2025pydismech} subsequently released Py-DiSMech, a vectorized Python reimplementation that further improves computational efficiency while adding PI-based shape control and modular actuation interfaces.

\section{Neural Surrogate Models}
\label{sec:surrogate}

Reinforcement learning for soft robots requires millions of environment interactions, yet first-principles Cosserat rod simulators are expensive (\Cref{sec:system:backends}).
Neural surrogate models address this bottleneck by replacing the physics solver with a learned transition function $f_\theta$ that approximates the state-to-state mapping in a single GPU forward pass (${\sim}0.001$\,ms), targeting four-to-five orders-of-magnitude speedup.
This section presents the implemented data-driven surrogate and outlines two physics-informed extensions --- a physics regularizer and a domain-decoupled PINN --- as proposed future directions.

\subsection{MLP Surrogate}
\label{sec:surrogate:model}

\subsubsection{Input Representation}
\label{sec:surrogate:model:input}

The surrogate input concatenates three components:
\begin{equation}
  \mathbf{z}_t
    = \bigl[\bar{\mathbf{s}}_t \;\|\; \mathbf{a}_t \;\|\;
            \boldsymbol{\phi}_1, \ldots, \boldsymbol{\phi}_{N_e}\bigr]
    \in \mathbb{R}^{189}
  \label{eq:surrogate-input}
\end{equation}

\begin{table}[H]
  \centering
  \small
  \caption{Surrogate input decomposition ($189 = 124 + 5 + 60$).}
  \label{tab:surrogate-input}
  \begin{tabular}{lrl}
    \toprule
    Component & Dim & Description \\
    \midrule
    $\bar{\mathbf{s}}_t$ & 124 & Z-score normalized rod state \\
    $\mathbf{a}_t$ & 5 & Normalized CPG action $\in [-1,1]^5$ \\
    $\boldsymbol{\phi}_1, \ldots, \boldsymbol{\phi}_{20}$ & 60 & Per-element phase triplets \\
    \bottomrule
  \end{tabular}
\end{table}

\subsubsection{Per-Element CPG Phase Encoding}
\label{sec:surrogate:model:phase}

For element $e$, the phase triplet is:
\begin{equation}
  \boldsymbol{\phi}_e
    = \bigl(\sin(2\pi f \cdot t_e),\;
            \cos(2\pi f \cdot t_e),\;
            \kappa_e^{\text{actual}}\bigr), \qquad
  t_e = \frac{k \cdot s_e}{f}
  \label{eq:per-element-phase}
\end{equation}

This provides the exact phase of the serpenoid wave at each element's location, plus the instantaneous curvature $\kappa_e^{\text{actual}}$ for the control error.
Phase features are computed on-the-fly from stored action and $t_{\text{start}}$.

An earlier V1 global phase encoding ($\mathbb{R}^2$) achieved only $R^2 = 0.23$ for angular velocity; the V2 per-element encoding significantly improved prediction.

\subsubsection{Architecture and Prediction}
\label{sec:surrogate:model:arch}

The surrogate predicts a normalized state delta:
\begin{equation}
  f_\theta(\mathbf{z}_t) \approx T(\mathbf{s}_t, \mathbf{a}_t) - \mathbf{s}_t, \qquad
  \hat{\mathbf{s}}_{t+1}
    = \mathbf{s}_t
      + \boldsymbol{\sigma}_\Delta \odot f_\theta(\mathbf{z}_t)
      + \boldsymbol{\mu}_\Delta
  \label{eq:next-state-reconstruction}
\end{equation}

Network architecture:
\begin{equation}
  \mathbb{R}^{189}
    \xrightarrow{\text{Linear}}
    \bigl[\text{Linear}(512) \to \text{LayerNorm} \to \text{SiLU}\bigr]^{\times 3}
    \xrightarrow{\text{Linear}}
    \mathbb{R}^{124}
  \label{eq:architecture}
\end{equation}

\begin{itemize}
  \item Output layer initialized to zero (default prediction: mean empirical transition)
  \item Total trainable parameters: ${\sim}659$K
  \item Delta prediction concentrates the target distribution and provides an implicit residual connection
\end{itemize}

\subsubsection{Training}
\label{sec:surrogate:model:training}

The surrogate is trained with a single-step MSE loss on normalized state deltas:
\begin{equation}
  \mathcal{L}_{\text{single}} = \frac{1}{N} \sum_{i=1}^{N} \bigl\| f_\theta(\mathbf{z}_t^{(i)}) - \overline{\Delta\mathbf{s}}_{\text{true}}^{(i)} \bigr\|^2
  \label{eq:surrogate-loss}
\end{equation}

\paragraph{Density-weighted sampling.}
Each training sample is weighted by inverse estimated local density:
\begin{equation}
  w^{(i)} = \frac{1}{\hat{p}(\mathbf{c}^{(i)})}, \qquad
  \mathbf{c}^{(i)} = \bigl[\bar{x}_{\text{CoM}},\; \bar{y}_{\text{CoM}},\; \|\dot{\mathbf{x}}\|,\; |\bar{\omega}_z|\bigr]
  \label{eq:density-weight}
\end{equation}
Weights are clipped at $10\times$ the mean to prevent rare samples from dominating.

% ---- 4.4 Physics-Informed Neural Networks (Proposed) ----
\subsection{Physics-Informed Extensions (Proposed)}
\label{sec:pinn}

Two physics-informed surrogate extensions are proposed as future work but have not been implemented: a \emph{physics regularizer} that adds soft algebraic constraints (kinematics, curvature--moment consistency, bounded energy) to the data-driven loss, and a \emph{domain-decoupled PINN} (DD-PINN) \citep{Krauss2024,Licher2025} that removes time as a network input, instead outputting parameters of an analytically differentiable ansatz --- yielding exact initial conditions, closed-form gradients (no autodiff), and $5$--$38\times$ faster training than standard PINNs.
Full formulations, loss functions, architectural details, and a four-way comparison (PDE solver vs.\ data-driven vs.\ PINN vs.\ DD-PINN) are given in \Cref{app:pinn}.

% ---- 4.5 Surrogate Comparison ----
\subsection{Surrogate Comparison}
\label{sec:pinn:results}

The data-driven surrogate achieves highest throughput and simplest training but provides no physics guarantees.
The physics-regularized variant would add soft constraints at minimal overhead (${\sim}0.1$\,ms GPU).
The DD-PINN would satisfy initial conditions exactly, eliminate autodiff cost, and enforce governing equations --- at the cost of ansatz design.
A detailed four-way comparison is provided in \Cref{app:pinn}.
Implementing and benchmarking these two physics-informed extensions is a priority for future work.

\subsubsection{Challenges}
\label{sec:rl-elastica:challenges}

\begin{itemize}
  \item \textbf{Computational cost:} ${\sim}46$\,ms/step $\times$ 16 workers $= {\sim}350$ steps/s; $2 \times 10^6$ steps requires ${\sim}36$\,h. DisMech is ${\sim}4\times$ faster but still CPU-bound.
  \item \textbf{Exploration in CPG space:} Large regions produce no locomotion (zero amplitude, extreme frequencies). Curriculum learning over parameter subsets improves efficiency.
  \item \textbf{Reward sparsity:} Distance-based rewards require discovering forward-moving gaits before receiving signal.
  \item \textbf{Backend discrepancy:} DisMech-trained policies do not transfer to Elastica due to friction, curvature mechanism, and temporal resolution differences.
\end{itemize}

\paragraph{Replication adjustments.}
Two configuration errors in our initial DisMech replication were identified by comparison with the paper's Table~A.3.
The rod radius was set to $r = 0.001$\,m (50$\times$ too thin) and Young's modulus to $E = 2$\,MPa (5$\times$ too soft), producing a near-limp structure that capped reward at ${\sim}$31.

% ---- 5.3 Reward Variants and Initial Ablation ----
\subsection{Reward Variants and Initial Ablation (E2)}
\label{sec:rl-dismech:ablation}

\subsubsection{Reward Variants}
\label{sec:rl-dismech:reward}

\citet{Choi2025} define the follow-target reward with two terms:
\begin{equation}
  r(s, s') = \underbrace{\exp\!\bigl(-5\,d(s')\bigr)}_{r_{\text{dist}}} + \underbrace{10\,\bigl(d(s) - d(s')\bigr)}_{r_{\text{improve}}}, \qquad d(s) = \lVert \mathbf{x}_{\text{tip}} - \mathbf{x}_{\text{target}} \rVert.
  \label{eq:reward-original}
\end{equation}
The first term is a bounded distance reward in $[0, 1]$, approaching 1 as the tip reaches the target.
The second is an improvement bonus proportional to the step-wise distance reduction.
The scaling factor of 5 was chosen to match the workspace radius; the improvement multiplier of 10 amplifies the approach signal.

\paragraph{Simplification for PPO.}
The original reward was designed for SAC, which averages over a replay buffer and tolerates high reward variance.
For on-policy PPO, the improvement bonus with its $10{\times}$ multiplier creates a reward SNR of only 0.08 (the signal is 8\% of the noise), causing the value function to fail.
We therefore use only the distance component as the base reward:
\begin{equation}
  r_{\text{dist}}(s) = \exp\!\bigl(-5\,d(s)\bigr).
  \label{eq:reward-dist}
\end{equation}
This raised the SNR from 0.08 to 1.02 (a $13{\times}$ improvement) and stabilized explained variance from 0.24 to 0.65.
We then investigate three augmentations to this simplified base reward, described below.

\paragraph{Heading reward.}
A cosine-similarity bonus for pointing the tip toward the target:
\begin{equation}
  r_{\text{heading}}(s) = \frac{1 + \cos\theta}{2}, \qquad
  r_{\text{total}} = (1-w)\,r_{\text{dist}} + w\,r_{\text{heading}},
  \label{eq:reward-heading}
\end{equation}
where $\theta$ is the angle between the tip tangent and the tip-to-target direction, and $w \in [0,1]$ is a mixing weight.
Both components lie in $[0,1]$, so the total reward is bounded.

\paragraph{Potential-Based Reward Shaping (PBRS).}
Following \citet{Ng1999} (with the equivalence to Q-value initialization established by \citet{Wiewiora2003}), we define a potential $\Phi(s) = -d(s)$ and add the shaping reward:
\begin{equation}
  F(s, s') = \gamma_{\text{pbrs}}\,\Phi(s') - \Phi(s) = d(s) - \gamma_{\text{pbrs}}\,d(s'),
  \label{eq:pbrs}
\end{equation}
where $\gamma_{\text{pbrs}}$ is a discount factor (we use 0.99).
This is guaranteed not to change the optimal policy.
When combined with the base reward: $r_{\text{total}} = r_{\text{dist}} + F(s,s')$.

\paragraph{PBRS-only.}
An ablation where the base reward is removed entirely, using only the shaping signal: $r_{\text{total}} = F(s,s')$.

\paragraph{Curriculum learning.}
All runs use curriculum learning: target speed ramps from 20\% to 100\% over the first 100 episodes per worker, avoiding reward-scale shock at initialization.

\paragraph{Fair comparison metric.}
Different reward functions produce incomparable total rewards (e.g., heading inflates the total by up to 72\%).
We log per-component rewards and use $r_{\text{dist}} = \exp(-5d)$ as the \emph{fair metric} for cross-run comparison, since it depends only on the physical outcome (distance to target) and is independent of the reward function.

\paragraph{Per-step SNR in PBRS.}
PBRS exhibits a structural signal-to-noise problem when used without a base reward:

\begin{itemize}
  \item \textbf{Root cause:} $F(s,s') = \gamma\,\Phi(s') - \Phi(s)$ differences consecutive potentials with similar values---analogous to noisy numerical differentiation.
  \item \textbf{Episode vs.\ step:} Telescoping gives healthy episode-level SNR (${\approx}\,3.3$), but per-step SNR collapses (${\approx}\,0.04$).
  \item \textbf{Feedback loop:} Per-step noise $\to$ critic overfits noisy value landscape $\to$ escalating gradients ($1.1 \to 15.5$) $\to$ noisier advantages.
  \item \textbf{Conclusion:} PBRS is an \emph{additive supplement} to a base reward~\citep{Ng1999,Wiewiora2003}, not a standalone signal. The base $\bar{r}_{\text{dist}} = \exp(-5d)$ provides a smooth anchor for the critic; PBRS accelerates credit assignment on top. Recent work by \citet{Mueller2025} formalizes how a linear shift of the potential function can improve PBRS effectiveness without sacrificing policy invariance.
\end{itemize}

\noindent
Standard mitigations---reward normalization (std-only, preserving the PBRS zero-mean property), higher GAE~$\lambda$ ($0.97$--$0.99$), gradient clipping at $0.5$, and advantage normalization---address the remaining variance.
More aggressive approaches ($k$-step PBRS, EMA-smoothed potentials) sacrifice strict policy-invariance and remain unvalidated for this domain.

  Subsequent training shows no improvement; the rolling-100 reward oscillates between 1 and 57 with no upward trend.

  Entropy values of 0.03--0.08 (vs.\ an initial entropy coefficient of 0.1) indicate the policy has narrowed its exploration.

  The critic loss swings from $10^{-3}$ to ${\sim}4.6$, indicating the value function struggles to fit the return distribution.

  \item \textbf{Mean distance to goal stagnates at ${\sim}$0.6--0.7.}
  The policy learns to move generally toward the target but cannot close the remaining distance.
  This matches the reachability diagnostic (\Cref{tab:reachability}): the reward gradient at $d \approx 0.6$ is $\exp(-3.0) \approx 0.05$, providing negligible learning signal.
\end{itemize}

\noindent
The extended training confirms that the 5M-frame plateau is a fundamental limit of the $\exp(-5d)$ reward, not a sample-efficiency issue.
Simply training longer does not help because the reward landscape itself provides insufficient gradient signal at the operating distances.
This motivates the normalized reward architecture in \Cref{sec:reward}.

% ---- 5.6 SAC vs PPO Stability ----
\subsection{SAC vs.\ PPO Stability (E5)}
\label{sec:rl-dismech:sac}

In separate experiments, we trained SAC (20M frames, 100 envs) and PPO (50M frames, 100 envs) on the same task using the paper-matching 3$\times$256 ReLU network.
SAC reached a best reward of 73.84 early (${\sim}$2M frames) but was unable to sustain performance, while PPO plateaued at ${\sim}$17 average reward.
\Cref{fig:choi2025-grad-norms} reveals the root cause: without entropy regularization ($\alpha=0$) and without gradient clipping, SAC's actor gradient norm grows from ${\sim}$0.04 to ${\sim}1.5 \times 10^{10}$ over 20M frames, while the critic gradient norm remains bounded at ${\sim}$1--40.
PPO gradient norms stay bounded at 2--28 throughout, consistent with its clipped surrogate objective.

  \centering
  \includegraphics[width=\linewidth]{choi2025_grad_norms.pdf}
  \caption{Gradient norms for SAC (left, log scale) and PPO (right, linear scale). SAC actor gradients explode by 10 orders of magnitude; PPO gradients remain stable.}

The paper's 5M frame budget may mask this instability, as the collapse onset occurs at ${\sim}$3--5M frames.
Possible mitigations include mild entropy regularization ($\alpha > 0$), actor-specific gradient clipping, or learning rate scheduling.

% ---- 5.6 Normalized Reward Architecture ----
\subsection{Normalized Reward Architecture}
\label{sec:reward}

The preceding experiments (E2--E5) reveal that reward design and curriculum learning dominate performance, outweighing both sample budget and algorithm choice.
The ablation (E2) identified three structural problems with ad-hoc reward design for on-policy PPO: high reward variance from scale-entangled components, auxiliary objectives dominating the total reward, and single-objective shaping missing the heading dimension.
The extended training (E4) confirmed these are fundamental limitations, not sample-budget issues.
This subsection presents a principled reward architecture that addresses all three by normalizing components to known ranges before weighting and decomposing PBRS into per-objective potentials.

\subsubsection{Problems with Ad-Hoc Design}
\label{sec:reward:problems}

\subsubsection{Architecture}
\label{sec:reward:arch}

The reward separates base components (normalized, then weighted) from PBRS components (raw, unweighted):
\begin{equation}
  r(s, a, s') = \underbrace{\sum_{i} w_i \cdot \bar{r}_i(s, a, s')}_{\text{base (normalized, weighted)}}
              + \underbrace{\sum_{j} \bigl(\gamma_{\text{pbrs}}\,\Phi_j(s') - \Phi_j(s)\bigr)}_{\text{PBRS (raw, unweighted)}}.
  \label{eq:reward-normalized}
\end{equation}
Each base component $\bar{r}_i$ is mapped to a known range \emph{before} weighting, so doubling $w_i$ exactly doubles the contribution of component~$i$ without interacting with scale.
The PBRS terms follow \citet{Ng1999}: $F_j(s,s') = \gamma_{\text{pbrs}}\,\Phi_j(s') - \Phi_j(s)$ is guaranteed to preserve the optimal policy.
This multi-component, hierarchical structure is consistent with the HPRS framework of \citet{Berducci2022}, which formalizes potential-based shaping over partially-ordered task requirements while preserving optimality.

\paragraph{Base components.}
\Cref{tab:reward-base} defines the three normalized base components.

\begin{table}[H]
  \centering
  \small
  \renewcommand{\arraystretch}{1.3}
  \caption{Normalized base reward components. All are mapped to known ranges before weighting.}
  \label{tab:reward-base}
  \begin{tabular}{@{}llcl@{}}
    \toprule
    \textbf{Component} & \textbf{Formula} & \textbf{Range} & \textbf{Weight} \\
    \midrule
    Distance    & $\bar{r}_{\text{dist}} = \exp(-5\,d)$                              & $[0, 1]$  & $w_{\text{dist}} = 1.0$ (always on) \\
    Heading     & $\bar{r}_{\text{heading}} = (1 + \cos\theta) / 2$                  & $[0, 1]$  & $w_{\text{heading}} \in \{0, 0.3\}$ \\
    Smoothness  & $\bar{r}_{\text{smooth}} = -\lVert\Delta a\rVert^2 / (2\,n_a)$     & $[-1, 0]$ & $w_{\text{smooth}} \in \{0, 0.01\text{--}0.05\}$ \\
    \bottomrule
  \end{tabular}
\end{table}

\noindent
Here $\theta$ is the angle between the tip tangent and the tip-to-target direction, $\Delta a = a_t - a_{t-1}$, and $n_a$ is the action dimensionality.
The smoothness bound follows from $\lVert\Delta a\rVert^2 \leq 2\,n_a$ when each action coordinate lies in $[-1,1]$.

\paragraph{PBRS components.}
\Cref{tab:reward-pbrs} defines the per-objective potentials.

\begin{table}[H]
  \centering
  \small
  \renewcommand{\arraystretch}{1.3}
  \caption{PBRS potentials. Each enters the reward as $F_j = \gamma_{\text{pbrs}}\,\Phi_j(s') - \Phi_j(s)$, unweighted.}
  \label{tab:reward-pbrs}
  \begin{tabular}{@{}llcl@{}}
    \toprule
    \textbf{Potential} & \textbf{Formula} & \textbf{Range} & \textbf{Role} \\
    \midrule
    Distance & $\Phi_{\text{dist}}(s) = -d(s) / R_{\text{ws}}$  & $[-1, 0]$ & Approach signal (replaces improvement bonus) \\
    Heading  & $\Phi_{\text{heading}}(s) = \cos\theta(s)$        & $[-1, 1]$ & Orientation improvement toward target \\
    \bottomrule
  \end{tabular}
\end{table}

\noindent
Normalizing $\Phi_{\text{dist}}$ by the workspace radius $R_{\text{ws}}$ keeps the shaping magnitude independent of workspace scale.
The heading potential rewards the agent for \emph{turning toward} the target (change in alignment), complementing the base heading component which rewards \emph{being aligned} (instantaneous state).
The telescoping property ensures the cumulative shaping over an episode reduces to a boundary term: $\sum_{t=0}^{T-1} F_j(s_t, s_{t+1}) = \gamma_{\text{pbrs}}\,\Phi_j(s_T) - \Phi_j(s_0)$ when $\gamma_{\text{pbrs}} = 1$.

\subsection{Steepness and Environment Scaling (E7)}
\label{sec:reward:steepness}

The reward landscape diagnostics (\Cref{sec:rl-dismech:diagnostics}) showed that the $\exp(-5d)$ reward is extremely sparse at typical operating distances, with shallower alternatives such as $\exp(-2d)$ or $\exp(-d)$ providing 5--$10\times$ denser signal (\Cref{tab:reward-landscape}).
This naturally suggests that reducing the steepness coefficient $k$ in $\exp(-k\,d)$ could break the distance plateau by providing a stronger gradient at mid-range distances.
We tested this hypothesis directly.

\paragraph{Steepness sweep.}
Using the best reward stack from \Cref{sec:rl-dismech:ablation-v2} (C5: $w_{\text{dist}} = 0.3$, $\gamma_{\text{pbrs}} = 0.99$, $w_{\text{smooth}} = 0.02$, curriculum with 500-episode warmup), we swept the steepness coefficient $k \in \{1, 2, 3, 5\}$ with 100 parallel environments and a 90-minute wall-time budget (${\sim}$10.5M frames per run).
\Cref{tab:steepness-results} shows the results.

The results disprove the hypothesis.
The paper default $k{=}5$ achieves the lowest distance despite providing the sparsest reward signal (column~3 shows the reward at $d{=}0.6$, a typical operating distance).
Shallower steepness causes \emph{reward hacking}: at $k{=}1$, $\exp(-0.6) = 0.549$---the agent receives 55\% of the maximum reward without approaching the target, eliminating approach pressure.
This manifests as negative explained variance ($\text{EV} = -0.06$) and near-collapsed action diversity ($\sigma_{\min} = 0.069$): the critic cannot learn a useful value landscape because all states look similarly rewarding.

The failure mode worsens monotonically: as $k$ decreases from 5 to 1, Q4 distance increases (0.675 $\to$ 0.725 $\to$ 0.790 $\to$ 0.819), and $\Delta$Dist turns positive (distance \emph{increasing} over training).
The shallower variants converge to a ``lazy equilibrium'' where hovering at medium distance yields high reward with low effort.

\label{sec:reward:eval}

\paragraph{Analysis: the train--eval gap.}
The training best reward of 90.20 corresponds to a single batch mean, not a robust evaluation.
PPO logs \texttt{best\_reward} as the maximum of \texttt{mean\_episode\_reward} across all batches; with 100 environments contributing to each batch, a single favorable batch---where most environments happen to start near the target---can produce an outlier reward that never recurs.

Inspecting the training trajectory confirms this: the mean reward oscillates between 0 and 32 across batches (final batch: 32.41), with \texttt{best\_reward} never improving after batch~450 (66 batches of stagnation).
The mean distance at the best-reward batch was 0.484\,m, but the final training distance was 0.776\,m---the policy did not sustain the improvement.

Three factors explain the gap:

\begin{enumerate}
  \item \textbf{Curriculum confound.}
  Training used curriculum learning with a 100-episode warmup that ramps target speed from 20\% to 100\%.
  Early episodes face near-stationary targets, inflating rewards.
  Evaluation uses the full target speed from the start, so the policy faces harder tracking.

  \item \textbf{Sparse reward plateau.}
  At the evaluation mean distance of $d{=}0.795$\,m, the reward signal $\exp(-5 \times 0.795) = 0.019$ is nearly zero.
  The policy has learned to move in the general direction of targets but lacks the fine-grained gradient to close the final 0.5--0.8\,m gap.

  \item \textbf{Stochastic vs.\ deterministic policy.}
  Training uses a stochastic policy that occasionally stumbles into high-reward states via exploration noise.
  Deterministic evaluation removes this, exposing that the \emph{learned mean} has not converged to an effective tracking strategy.
\end{enumerate}

This visualization was recorded from the best checkpoint with three episodes at 20\,FPS.

\paragraph{Implications.}
The evaluation confirms that the $d \approx 0.6$--$0.8$ distance plateau observed during training (\Cref{sec:reward:steepness}) is a genuine performance ceiling, not a logging artifact.
The best training reward is a poor proxy for policy quality in this domain---future work should use rolling evaluation windows and deterministic evaluation checkpointing rather than single-batch best reward.

\noindent
The DisMech ablations reveal three critical insights: (1)~the $\exp(-5d)$ reward is fundamentally sparse---70\% of steps yield reward ${<}\,0.05$---creating a hard plateau at $d \approx 0.6$ that 50M frames cannot overcome; (2)~curriculum learning, not reward shaping, is the primary mechanism for breaking this plateau; (3)~reducing steepness to provide ``denser'' signal backfires via reward hacking---the agent earns high reward at medium distance without approaching, a cautionary result against na\"ive reward densification.
SAC exhibits gradient instability beyond ${\sim}$3--5M frames without entropy regularization.

\paragraph{Limitations.}
Several factors constrain the generality of these results.

\begin{enumerate}
  \item \textbf{Single seed and fixed architecture.}
  All RL experiments use a single seed (42) and a fixed network architecture ($3 \times 256$ ReLU MLP); results may not hold under different initializations or capacity regimes.

  \item \textbf{Surrogate velocity fidelity.}
  The neural surrogate achieves strong CoM prediction ($R^2 = 0.852$) but poor velocity prediction ($R^2 = 0.145$), making multi-step rollout fidelity uncertain --- surrogate-based RL training has not yet been attempted.

  \item \textbf{Planar (2D) physics only.}
  Both physics backends are planar; extending to 3D Cosserat rods would increase the state dimensionality from 124 to ${\sim}$370, likely affecting both surrogate accuracy and RL sample efficiency.

  \item \textbf{Limited ablation budget.}
  The normalized reward ablation uses a 30-minute wall-time budget (${\sim}$3.6M frames per run); longer training horizons may change the relative ranking of configurations, as the initial ablation showed that curriculum effects compound over time.

  \item \textbf{No cross-backend transfer.}
  Policies trained on one backend do not transfer to the other due to differences in friction models, curvature mechanisms, and temporal resolution.
\end{enumerate}

\paragraph{Future work.}
Several directions emerge from this work.

\begin{enumerate}
  \item \textbf{Physics-informed surrogates.}
  The DD-PINN formulation (\Cref{sec:pinn}) would enforce governing equations at collocation points while satisfying initial conditions exactly, potentially improving velocity prediction without the autodiff overhead of standard PINNs.
  Benchmarking the DD-PINN against the data-driven baseline is the immediate next step for the surrogate pipeline.

  \item \textbf{Surrogate-accelerated RL.}
  Once surrogate fidelity is sufficient, the ${\sim}$46\,ms-to-${\sim}$0.001\,ms speedup would enable training at $10^4$--$10^5 \times$ the current throughput.
  A sim-to-sim transfer protocol --- pre-train on surrogate, fine-tune on simulator --- could overcome the fidelity gap.

  \item \textbf{Hierarchical control.}
  The current work trains flat policies for the approach phase.
  The full HRL pipeline --- a meta-controller selecting between approach and coil sub-policies --- remains to be integrated.
  The normalized reward architecture is designed to extend to the coil phase by adding task-specific base components (e.g., contact area, wrap angle) while preserving the weight-decoupled structure.

  \item \textbf{SAC stabilization.}
  SAC's early reward peak (73.84 at ${\sim}$2M frames, vs.\ PPO's 65.81 at ${\sim}$12M) suggests higher sample efficiency if the gradient instability can be controlled.
  Entropy regularization ($\alpha > 0$), actor-specific gradient clipping, and learning rate scheduling are immediate candidates.

  \item \textbf{Pipeline inspection navigation.}
  A natural industrial application is autonomous pipeline inspection, where the snake robot navigates through confined tunnel networks to locate sections requiring maintenance or repair.
  This scenario introduces additional challenges beyond open-field predation: constrained geometry requiring collision-aware locomotion, branching decision-making at junctions, and anomaly detection (e.g., corrosion, leaks, structural deformation) as a secondary objective alongside navigation.
  The hierarchical control framework developed in this work could be extended with an exploration-level meta-controller that plans routes through the pipe network while low-level policies handle locomotion through straight segments, bends, and junctions.
\end{enumerate}

\paragraph{Closing remarks.}
This work demonstrates that reward design and curriculum learning are the primary levers for RL-based snake robot control, outweighing both algorithmic choice (PPO vs.\ SAC) and sample budget ($10 \times$ more frames does not overcome a sparse reward).
The normalized reward architecture provides a principled framework for composing multiple objectives without scale entanglement, and the per-component logging methodology enables early detection of reward hacking --- a failure mode that aggregate metrics alone cannot reveal.
These findings lay the groundwork for hierarchical predation control, where the same architectural principles can be extended to multi-phase, multi-objective reward design.

\subsection{Method of Lines and Explicit Integration}
\label{app:physics:mol}

The PDE-to-ODE reduction follows two stages:

\paragraph{Stage 1 --- Spatial discretization (method of lines).}
All spatial derivatives $\partial/\partial s$ are replaced by sparse matrix--vector products:
\begin{itemize}
  \item Force gradient: $\partial \mathbf{F}/\partial s \to \mathbf{D}^T \mathbf{F}_e$ (bidiagonal, $O(N)$)
  \item Strain from geometry: $\boldsymbol{\varepsilon}_e = \mathbf{Q}_e^T (\mathbf{x}_{e+1} - \mathbf{x}_e)/\ell_e^{\text{rest}}$
  \item Constitutive law: $\mathbf{F}_e = \mathbf{S}(\boldsymbol{\varepsilon}_e - \boldsymbol{\varepsilon}_e^{\text{rest}})$ (pointwise)
\end{itemize}
After replacement, the PDE becomes the ODE $\dot{\mathbf{s}} = \mathbf{g}(\mathbf{s}, \mathbf{a}, t)$ with $\mathbf{g}$ a closed-form function.

\paragraph{Stage 2 --- Explicit time integration.}
An explicit integrator evaluates $\mathbf{g}$ at the current state and writes the result directly:
$\mathbf{s}^{n+1} = \Phi(\mathbf{s}^n, \Delta t)$.
The unknown appears only on the left-hand side --- no equation to solve.

\paragraph{CFL stability constraint.}
$\Delta t < C\sqrt{\rho A \Delta s^4 / (EI)}$, giving $\Delta t_{\max} \approx 1.7 \times 10^{-3}$\,s.
PyElastica uses $\Delta t = 10^{-3}$\,s (safely below), forcing $n_{\text{sub}} = 500$.

\subsection{How Spatial Discretization Converts PDEs to ODEs}
\label{app:physics:discretization-intuition}

This section provides an intuitive explanation of the two-stage PDE-to-ODE reduction summarized in \Cref{app:physics:mol}.

Recall that nodes and elements live at \emph{different} locations on the rod:

\noindent
\textbf{Nodes} ($N{=}21$) are at the physical vertices; they store translational DOFs $(x_i, y_i, \dot{x}_i, \dot{y}_i)$.
\textbf{Elements} ($N_e{=}20$) are the segments between adjacent nodes; they store rotational DOFs $(\psi_e, \omega_{z,e})$.
The difference operators $\mathbf{D}$ and $\mathbf{D}^T$ transfer information between these two layers.

\subsubsection{The Difference Matrix $\mathbf{D}$ (Node $\to$ Element)}

$\mathbf{D}$ is a $20 \times 21$ bidiagonal matrix that computes differences between adjacent nodes.
For element~$e$:
\begin{equation}
  (\mathbf{D}\,\mathbf{x})_e = x_{e+1} - x_e
  \label{eq:D-operator}
\end{equation}
Explicitly:
\begin{equation}
  \mathbf{D} =
  \begin{pmatrix}
    -1 & +1 &  0 &  0 & \cdots \\
     0 & -1 & +1 &  0 & \cdots \\
     0 &  0 & -1 & +1 & \cdots \\
       &    &    & \ddots & \ddots
  \end{pmatrix}
  \in \mathbb{R}^{20 \times 21}
  \label{eq:D-matrix}
\end{equation}
This approximates $\partial \mathbf{x}/\partial s$: the spatial derivative of position along the rod becomes a finite difference between adjacent nodes.
Dividing by segment rest length $\ell_e^{\text{rest}}$ yields the actual derivative; the matrix itself performs only the differencing.

\paragraph{Physical meaning.}
$\mathbf{D}\,\mathbf{x}$ gives the \emph{tangent vector} of each element---the direction and length of each segment.
This is required for the cross-product term $(\partial\mathbf{x}/\partial s) \times \mathbf{F}$ in the angular momentum equation~\eqref{eq:pde-angular}.

\subsubsection{The Transpose $\mathbf{D}^T$ (Element $\to$ Node)}

$\mathbf{D}^T$ is the transpose---a $21 \times 20$ matrix that maps element quantities back to nodes.
For node~$i$:
\begin{equation}
  (\mathbf{D}^T \mathbf{F})_i = F_i - F_{i-1}
  \label{eq:DT-operator}
\end{equation}
with $F_{-1} = F_{20} = 0$ at the free boundaries.
Explicitly:
\begin{equation}
  \mathbf{D}^T =
  \begin{pmatrix}
    -1 &  0 &  0 & \cdots \\
    +1 & -1 &  0 & \cdots \\
     0 & +1 & -1 & \cdots \\
     0 &  0 & +1 & \cdots \\
       &    &    & \ddots
  \end{pmatrix}
  \in \mathbb{R}^{21 \times 20}
  \label{eq:DT-matrix}
\end{equation}

\paragraph{Physical meaning.}
$\mathbf{D}^T \mathbf{F}$ computes the \emph{net force on each node} from its two neighboring elements.
In the continuous PDE, $\partial \mathbf{F}/\partial s$ gives the net force per unit length at a point.
In the discrete version, $\mathbf{D}^T \mathbf{F}_e$ gives the net force on each node:
interior node~$i$ receives $F_i - F_{i-1}$ (force from the element ahead minus the element behind),
while boundary nodes have only one adjacent element.

\paragraph{Concrete example.}
Consider 4 nodes with $x$-positions $[0.0,\; 0.25,\; 0.55,\; 1.0]$:
\begin{align}
  \mathbf{D}\,\mathbf{x} &= [0.25,\; 0.30,\; 0.45] && \text{(element tangent lengths)} \\
  \intertext{If internal forces on those 3 elements are $\mathbf{F}_e = [10,\; 8,\; 12]$:}
  \mathbf{D}^T \mathbf{F} &= [-10,\; 10{-}8,\; 8{-}12,\; 12] = [-10,\; 2,\; -4,\; 12] && \text{(net force per node)}
\end{align}
Node~0 has only element~0 pulling on it; node~1 has elements~0 and~1 pulling in opposite directions.

\subsubsection{Adjoint Structure and Energy Conservation}

The transpose relationship is not coincidental.
$\mathbf{D}$ and $\mathbf{D}^T$ are \emph{adjoint} operators---the discrete analogue of the continuous identity where the gradient and divergence operators are adjoints:
\begin{equation}
  \langle \mathbf{D}\,\mathbf{x},\; \mathbf{F} \rangle_{\text{elements}}
  = \langle \mathbf{x},\; \mathbf{D}^T \mathbf{F} \rangle_{\text{nodes}}
  \label{eq:adjoint}
\end{equation}
This duality ensures \emph{discrete energy conservation}: the work done by forces on nodes is consistent with the strain energy stored in elements, preventing artificial energy gain or loss from the discretization.

\subsubsection{Stage 1: Eliminating Spatial Derivatives}

With these operators, every $\partial/\partial s$ in the governing PDEs is replaced by a matrix--vector product.
For example, the linear momentum PDE
\begin{equation}
  \rho A \frac{\partial \mathbf{v}}{\partial t}
  = \frac{\partial \mathbf{F}}{\partial s} + \mathbf{f}_{\text{ext}}
\end{equation}
becomes, at each node~$i$:
\begin{equation}
  \rho A \frac{d \mathbf{v}_i}{d t}
  = (\mathbf{D}^T \mathbf{F}_e)_i + \mathbf{f}_{\text{ext},i}
  \label{eq:node-ode}
\end{equation}
No spatial derivatives remain---only time derivatives and algebraic dependence on the discrete state.

\subsubsection{Stage 2: Reduction to First-Order Form}

The original PDEs are \emph{second-order} in time (they involve accelerations $\partial \mathbf{v}/\partial t$ and $\partial \boldsymbol{\omega}/\partial t$).
To obtain first-order ODEs, velocities are introduced as independent state variables:
\begin{alignat}{2}
  \dot{x}_i &= v_{x,i}       &\qquad& \text{(definition of velocity)} \label{eq:first-order-pos}\\
  \dot{v}_{x,i} &= \frac{1}{\rho A}\bigl[(\mathbf{D}^T \mathbf{F}_e)_i + f_{\text{ext},i}\bigr]
    &\qquad& \text{(from linear momentum PDE)} \label{eq:first-order-vel}\\[4pt]
  \dot{\psi}_e &= \omega_{z,e} &\qquad& \text{(definition of angular velocity)} \label{eq:first-order-angle}\\
  \dot{\omega}_{z,e} &= \frac{1}{\rho I}\bigl[(\mathbf{D}_v \boldsymbol{\tau}_v)_e + m_{\text{ext},e} + \ldots\bigr]
    &\qquad& \text{(from angular momentum PDE)} \label{eq:first-order-omega}
\end{alignat}
This doubles the number of variables (positions $+$ angles $\to$ positions $+$ angles $+$ velocities $+$ angular velocities) but makes every equation first-order.
The result is the 124-dimensional system $\dot{\mathbf{s}} = \mathbf{g}(\mathbf{s}, \mathbf{a}, t)$ from \Cref{eq:ode-system}, ready for explicit time integration.

\subsection{DisMech Formulation}
\label{app:physics:dismech}

DisMech uses Discrete Elastic Rods (DER) with implicit backward Euler integration.

\paragraph{Backward Euler formulation.}
\begin{subequations}
\label{eq:dismech-backward-euler}
\begin{align}
  \mathbf{v}_{n+1} &= \mathbf{v}_n + \Delta t\,\mathbf{M}^{-1}\mathbf{F}(\mathbf{q}_{n+1},\, \mathbf{v}_{n+1}), \\
  \mathbf{q}_{n+1} &= \mathbf{q}_n + \Delta t\,\mathbf{v}_{n+1},
\end{align}
\end{subequations}

\paragraph{Nonlinear residual and Newton solver.}
\begin{equation}
  \mathbf{g}(\mathbf{q}_{n+1})
    \coloneqq \mathbf{M}\bigl(\mathbf{q}_{n+1} - 2\mathbf{q}_n + \mathbf{q}_{n-1}\bigr)
    - \Delta t^2\,\mathbf{F}(\mathbf{q}_{n+1})
    = \mathbf{0}
  \label{eq:dismech-residual}
\end{equation}
solved iteratively: $\mathbf{q}^{(k+1)} = \mathbf{q}^{(k)} - \mathbf{J}^{-1}\mathbf{g}(\mathbf{q}^{(k)})$ with convergence criteria $\|\mathbf{g}\| < 10^{-4}$ or $\|\delta\mathbf{q}\| < 10^{-2}$, max 25 iterations.

\paragraph{Discrete curvature.}
\begin{equation}
  \boldsymbol{\kappa}_i
    = \frac{2\,\mathbf{e}_{i-1} \times \mathbf{e}_i}
           {|\mathbf{e}_{i-1}|\,|\mathbf{e}_i| + \mathbf{e}_{i-1} \cdot \mathbf{e}_i}
  \label{eq:dismech-curvature-binormal}
\end{equation}

\section{Physics-Informed Neural Network Details (Proposed)}
\label{app:pinn}

This appendix provides the full mathematical formulations for the physics-informed surrogate extensions proposed in \Cref{sec:pinn}.
None of these have been implemented; they constitute a concrete design for future work.

\subsection{Standard PINN}
\label{sec:pinn:standard}

Physics-informed neural networks \citep{Raissi2019} augment the data-driven loss with a physics residual derived from the governing equations.
Given the state-space model $\dot{\mathbf{x}}(t) = \fssm(\mathbf{x}(t), \mathbf{u}(t))$, a network $\mathbf{h}_\theta$ predicts
$\xhat_t = \mathbf{h}_\theta(\mathbf{x}_0, \mathbf{u}, t)$.
The PINN loss:
\begin{multline}
  \mathcal{L}(\theta)
    = \underbrace{\lambda_{\mathrm{IC}} \bigl\| \xhat_0 - \mathbf{x}_0 \bigr\|^2}_{\mathcal{L}_{\mathrm{IC}}}
    + \underbrace{\lambda_{\mathrm{phys}} \frac{1}{N_c}\sum_{k=1}^{N_c}
      \left\| \frac{\partial \xhat_{t_k}}{\partial t} - \fssm(\xhat_{t_k}, \mathbf{u}_{t_k}) \right\|^2}_{\mathcal{L}_{\mathrm{phys}}} \\
    + \underbrace{\lambda_{\mathrm{data}} \frac{1}{N_d}\sum_{k=1}^{N_d}
      \left\| \xhat_{t_k} - \mathbf{x}_{\mathrm{data},k} \right\|^2}_{\mathcal{L}_{\mathrm{data}}}
  \label{eq:pinn:loss}
\end{multline}

The time derivative $\partial \xhat_t / \partial t$ is obtained via autodiff through the network, doubling the computational cost per collocation point.

\paragraph{Practical bottlenecks:}
\begin{itemize}
  \item \textbf{Expensive autodiff} --- additional backward pass per collocation point
  \item \textbf{Loss balancing} --- three competing objectives require careful weighting
  \item \textbf{Training instability} --- can diverge for systems with $m \gg 1$ coupled states
\end{itemize}

\label{sec:pinn:regularizer}

The simplest physics-informed approach adds algebraic constraints as soft regularizers.
Four constraints are enforced via trapezoidal integration on the 124-dimensional state delta $\Delta\mathbf{x} = \mathbf{x}_{t+1} - \mathbf{x}_t$:

\subsection{DD-PINN Method}
\label{sec:pinn:ddpinn}

The domain-decoupled PINN \citep{Krauss2024,Licher2025} resolves all three PINN bottlenecks: \textbf{remove time as a network input} and output parameters of an analytically differentiable ansatz.

\begin{equation}
  \boxed{\xhat_t = \mathbf{g}(\avec, t) + \mathbf{x}_0, \qquad \avec = \fnn(\mathbf{x}_0, \mathbf{u}_0;\, \theta)}
  \label{eq:pinn:ddpinn-core}
\end{equation}

The ansatz for each state variable $j$:
\begin{equation}
  g_j(t) = \sum_{i=1}^{n_g} \alpha_{ij}\, e^{-\delta_{ij} t}
    \bigl[\phig(\beta_{ij}\, t + \gamma_{ij}) - \phig(\gamma_{ij})\bigr]
  \label{eq:pinn:ansatz}
\end{equation}
where $\phig(x) = \sin(x)$ and the subtraction ensures $g_j(0) = 0$ exactly.

\paragraph{Key advantages over standard PINN:}
\begin{itemize}
  \item \textbf{Exact IC:} $\mathbf{g}(\avec, 0) \equiv \mathbf{0}$ by construction
  \item \textbf{Closed-form gradients:} $\dot{\mathbf{g}}$ computed analytically --- no autodiff cost
  \item \textbf{Simplified loss:} only 2 terms ($\mathcal{L}_{\mathrm{phys}} + \mathcal{L}_{\mathrm{data}}$)
  \item \textbf{More collocation points:} $10$--$25\times$ more than standard PINNs due to cheap evaluation
\end{itemize}

\subsection{Limitations of PINNs}
\label{sec:pinn:limitations}

PINNs face fundamental limitations for complex dynamical systems:
\begin{itemize}
  \item \textbf{Spectral bias:} Networks learn low-frequency components first, making high-frequency CPG dynamics difficult.
  \item \textbf{Scaling to high dimensions:} Standard PINNs struggle for $m \gg 1$ (the snake robot has $m = 124$).
  \item \textbf{Discontinuous physics:} Contact and friction forces create non-smooth dynamics that PINNs cannot easily enforce.
  \item \textbf{Ansatz design:} DD-PINNs require an appropriate ansatz function designed a priori for the dynamics class.
\end{itemize}

\subsection{Differentiable Cosserat Rod Right-Hand Side}
\label{sec:pinn:cosserat-rhs}

A differentiable implementation of $\fssm$ computes $\dot{\mathbf{x}} = \fssm(\mathbf{x})$ for the 124-dimensional state vector using an inextensible rod approximation (bending + RFT friction, no stretching).
The right-hand side assembles six blocks:
\begin{equation}
  \fssm(\mathbf{x}) = \bigl(\dot{x}_i,\; \dot{y}_i,\; \dot{v}_{x,i},\; \dot{v}_{y,i},\; \dot{\psi}_e,\; \dot{\omega}_{z,e}\bigr) \in \mathbb{R}^{124}
  \label{eq:pinn:rhs-full}
\end{equation}

Key components:
\begin{itemize}
  \item \textbf{Kinematics:} $\dot{x}_i = v_{x,i}$, $\dot{\psi}_e = \omega_{z,e}$
  \item \textbf{Bending moments:} $M_j = EI(\psi_{j+1} - \psi_j)/\Delta\ell$
  \item \textbf{RFT friction:} $\mathbf{F}_i^{\text{fric}} = -c_t\mathbf{v}_{\text{tan},i} - c_n\mathbf{v}_{\text{norm},i}$ with $\varepsilon$-regularized tangents
  \item \textbf{Newton's 2nd law:} $\dot{v}_{x,i} = (F_{x,i}^{\text{fric}} + F_{x,i}^{\text{elastic}})/m_i$
\end{itemize}

\subsection{Physics-Regularized Loss}
\label{sec:pinn:reg-loss}

For the regularized surrogate:
\begin{equation}
  \mathcal{L}^{(n)}(\theta) = \mathcal{L}_{\mathrm{data}}(\theta) + \lambda\,\sigma_{\mathrm{reg}}(n)\,\mathcal{L}_{\mathrm{reg}}(\theta)
  \label{eq:pinn:reg-total-loss}
\end{equation}
with piecewise-linear warmup (20\% data-only).

\subsection{DD-PINN Loss and Training}
\label{sec:pinn:ddpinn-training}

The DD-PINN training loss at epoch $n$:
\begin{equation}
  \boxed{%
    \mathcal{L}^{(n)}(\theta) = w_{\mathrm{data}}^{(n)}\,\mathcal{L}_{\mathrm{data}}(\theta)
    + w_{\mathrm{phys}}^{(n)}\,\lambda\,\sigma(n)\,\mathcal{L}_{\mathrm{phys}}(\theta)
  }
  \label{eq:pinn:ddpinn-total}
\end{equation}
where $w_{\mathrm{data}}^{(n)}, w_{\mathrm{phys}}^{(n)}$ are ReLoBRaLo \citep{Bischof2022} adaptive weights, $\lambda$ is fixed physics strength, and $\sigma(n)$ is a curriculum schedule (15\% warmup, 30\% ramp, full physics at 45\%).
