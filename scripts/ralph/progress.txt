## Codebase Patterns
- Project root must be on `sys.path` for scripts in `script/` to import `aprx_model_elastica` and `locomotion_elastica` — add `sys.path.insert(0, PROJECT_ROOT)` at top
- Data collection uses `_multiprocess_collect()` from `aprx_model_elastica.collect_data` with `DataCollectionConfig` from `aprx_model_elastica.collect_config`
- Batch files saved as `batch_wWW_NNNN.pt` (multi-worker) with keys: states, actions, serpenoid_times, next_states, episode_ids, step_indices
- State dim = 124, Action dim = 5 (constants in `aprx_model_elastica/state.py`)
- Episode ID offset per worker = `worker_id * 10_000_000` to avoid collisions
- Pre-existing mypy errors in collect_data.py/env.py are expected — typecheck criterion means "no NEW errors in your file"
